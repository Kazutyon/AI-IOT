1. 標準化とは

標準化とは、技術や手順を共通化し、誰でも同じように利用できるようにすることです。

標準化の目的

  • 互換性の確保（例：USB、Wi-Fi）
  • 品質の向上
  • 安全性の確保
  • 国際的な取引の円滑化
  • 技術の普及促進

IT業界では、国際標準が特に重要です。

2. ISO / IEC（国際標準化機関）

世界的に最も有名な標準化団体です。

ISO（国際標準化機構）

工業製品やマネジメントシステムの国際規格を策定。

  • ISO 9001（品質マネジメント）
  • ISO 14001（環境マネジメント）
  • ISO/IEC 27001（情報セキュリティマネジメント：ISMS）

IEC（国際電気標準会議）

電気・電子技術の国際規格を策定。

ISO/IEC 共同規格

IT分野では ISO と IEC が共同で規格を作ることが多いです。

  • ISO/IEC 27001（ISMS）
  • ISO/IEC 12207（ソフトウェアライフサイクルプロセス）
  • ISO/IEC 25000（SQuaRE：ソフトウェア品質）

3. JIS（日本産業規格）

日本国内の標準規格で、ISO/IEC と整合性を持つものが多いです。

  • JIS Q 15001（個人情報保護マネジメント：Pマークの基準）
  • JIS X 25000（ソフトウェア品質モデル）

4. W3C（Web標準）

Web技術の標準化を行う国際団体です。WebブラウザやWebサイトが同じルールで動作するようにするための組織です。

主な標準

  • HTML
  • CSS
  • XML
  • Webアクセシビリティガイドライン（WCAG）

役割

  • Web技術の互換性を確保
  • Webの発展とオープン性を維持
  • アクセシビリティ（誰でも使えるWeb）の推進

5. IETF と RFC（インターネット標準）

インターネット技術の標準化を行う組織が IETF（Internet Engineering Task Force） です。

RFC（Request For Comments）とは

IETF が公開する技術文書で、インターネット技術の仕様書として使われます。

RFCで定義される主な技術

  • TCP / IP
  • HTTP
  • SMTP（メール）
  • DNS
  • IPアドレス

インターネットの基盤技術はほぼすべて RFC で定義されています。

7. 標準化と知的財産（FRAND）

標準化された技術には、特許が含まれる場合があります。その場合、特許権者は「FRAND条件」でライセンスを提供することが求められます。

FRANDとは

  • Fair（公正）
  • Reasonable（合理的）
  • Non-Discriminatory（非差別的）

標準技術を誰でも利用できるようにするためのルールです。

間違えやすいポイント

  • ISOとIECの役割を混同しない。 ISOは工業全般・マネジメント、IECは電気・電子技術が専門。IT分野は両者が共同（ISO/IEC）で規格策定することが多い。
  • W3CとIETFの役割を混同しない。 W3CはWeb技術（HTML・CSS）、IETFはインターネット基盤技術（TCP/IP・HTTP）を標準化。
  • RFCは「仕様書」であり「法律」ではない。 しかしインターネットの事実上の標準として広く採用されている。
  • FRANDは「無料」ではない。 公正・合理的・非差別的という条件でライセンス料を取ることができる。
  • JISはISO/IECと完全に同じではない。 日本国内向けに調整された規格であり、整合性は保たれているが細部が異なる場合がある。

確認問題

Q1. 標準化の目的として正しくないものはどれか。

ア）互換性の確保　イ）品質の向上　ウ）技術の独占　エ）国際的な取引の円滑化

答えを見る ▼

答え: ウ — 標準化の目的は技術の普及促進であり、独占ではない。

Q2. 情報セキュリティマネジメントシステム（ISMS）の国際規格はどれか。

ア）ISO 9001　イ）ISO 14001　ウ）ISO/IEC 27001　エ）ISO/IEC 25000

答えを見る ▼

答え: ウ — ISO/IEC 27001がISMSの国際規格である。

Q3. 電気・電子技術の国際規格を策定する団体はどれか。

ア）ISO　イ）IEC　ウ）W3C　エ）IETF

答えを見る ▼

答え: イ — IEC（国際電気標準会議）が電気・電子技術の国際規格を策定する。

Q4. Pマークの基準となっているJIS規格はどれか。

ア）JIS Q 15001　イ）JIS X 25000　ウ）JIS Q 27001　エ）JIS X 5080

答えを見る ▼

答え: ア — JIS Q 15001が個人情報保護マネジメント（Pマーク）の基準である。

Q5. W3Cが策定したWebアクセシビリティのガイドラインはどれか。

ア）HTML　イ）CSS　ウ）WCAG　エ）XML

答えを見る ▼

答え: ウ — WCAG（Web Content Accessibility Guidelines）がW3C策定のアクセシビリティガイドラインである。

Q6. IETFが公開するインターネット技術の仕様書を何というか。

ア）HTML　イ）RFC　ウ）ISO　エ）JIS

答えを見る ▼

答え: イ — RFC（Request For Comments）がIETFが公開する技術文書である。

Q7. RFCで定義されていない技術はどれか。

ア）TCP/IP　イ）HTTP　ウ）SMTP　エ）HTML

答えを見る ▼

答え: エ — HTMLはW3Cが策定するWeb標準であり、RFCで定義されるのはTCP/IP・HTTP・SMTPなどのインターネット基盤技術である。

Q8. 標準技術に含まれる特許のライセンス条件として求められるFRANDの意味として正しいものはどれか。

ア）無料で利用できること　イ）公正・合理的・非差別的であること　ウ）特定企業のみが利用できること　エ）特許権者が自由に条件を決められること

答えを見る ▼

答え: イ — FRANDはFair（公正）・Reasonable（合理的）・Non-Discriminatory（非差別的）の略である。

Q9. ソフトウェアライフサイクルプロセスの国際規格はどれか。

ア）ISO/IEC 12207　イ）ISO/IEC 27001　ウ）ISO/IEC 25000　エ）ISO 9001

答えを見る ▼

答え: ア — ISO/IEC 12207がソフトウェアライフサイクルプロセスの国際規格である。

Q10. W3Cの役割として正しいものはどれか。

ア）インターネットの基盤技術を標準化する　イ）Web技術の互換性を確保する　ウ）電気・電子技術の国際規格を策定する　エ）日本の産業規格を策定する

答えを見る ▼

答え: イ — W3CはWeb技術の標準化を行い、互換性を確保する団体である。
