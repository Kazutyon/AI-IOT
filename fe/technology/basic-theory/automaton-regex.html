形式言語の基礎概念

形式言語は「記号 → アルファベット → 文字列 → 言語」の4階層で構成されます。コンパイラの文法チェック・正規表現・通信プロトコル・入力バリデーションはすべてこの枠組みで動いています。

概念例（Σ = {0,1}）IT実務での対応
記号0, 1ソースコードの文字、バイト列
アルファベット{0, 1}文字セット（ASCII, UTF-8 など）
文字列01, 1010, 0011ソースコード、通信データ、入力値
言語{01を含む文字列の集合}文法を満たすプログラム、正しいHTTPリクエスト

規則 → 言語 → 判定 の流れ

正規表現（regular expression）

正規表現は「文字列の規則を短く・明確に・機械的に記述するための方法」です。FEでは記号の意味と読み方が出題されます。

記号意味例マッチする文字列例
*（スター）直前の要素が0回以上0*11, 01, 001, 0001 …
+（プラス）直前の要素が1回以上ab+ab, abb, abbb …（aのみはNG）
|（パイプ）どちらか一方（OR）01|1001 または 10
?（クエスチョン）直前の要素が0回または1回ab?a または ab
( )（括弧）グループ化(ab)+ab, abab, ababab …

正規表現の実務利用

  • 入力チェック（メールアドレス・電話番号の形式検証）
  • ログ解析（パターンに合う行を抽出）
  • プログラムの字句解析（コンパイラがトークンを分割）
  • セキュリティ（SQLインジェクション・不正入力の検出）

DFA（決定性有限オートマトン）

DFAはコンピュータが文字列を判定するための最もシンプルな機械モデルです。「有限個の状態」と「遷移関数」だけで動作します。

DFA の5要素

DFA = ( Q, Σ, δ, q₀, F )

  Q  : 状態の有限集合
  Σ  : アルファベット（入力記号の集合）
  δ  : 遷移関数  δ(q, a) = q′
  q₀ : 開始状態（q₀ ∈ Q）
  F  : 受理状態の集合（F ⊆ Q）
項目DFAでの役割人間の例え
状態 Q記憶の分類。有限個に限られる「偶数読んだ」「奇数読んだ」という記憶の種類
遷移関数 δ文字を読むたびに状態を更新する規則「1を読んだら偶奇を反転する」
開始状態 q₀文字列を読み始める最初の状態ゲームの初期位置
受理状態 F入力を読み終えたとき、ここにいれば受理ゴール地点（途中通過は無効）

DFAの「状態」とは何か

DFAの状態は「現在の物理的な状況」ではありません。これまでに読んだ入力列に関する抽象的な記憶です。判定に必要な最小限の情報だけを保持します。

  • 「1の出現回数が偶数か奇数か」→ 2状態で十分
  • 「最後に読んだ文字がaかbか」→ 2状態で十分
  • 「01 という並びが出現したか」→ 3状態で表現可能
  • 何文字読んだか・どの位置で1が出たか → 状態に保存する必要なし

状態遷移図と受理状態

FEでは状態遷移図を読んで「この文字列は受理されるか」を判定する問題が最頻出です。

アルファベット Σ = {0, 1}　開始状態: q0（1の出現回数 = 偶数）　受理状態: q0 ★

状態: q0 = これまでに読んだ「1」の回数が偶数 / q1 = これまでに読んだ「1」の回数が奇数

遷移規則: δ(q0, 0) = q0（0を読んでも偶奇変わらず） / δ(q0, 1) = q1（1を読んで奇数になる） / δ(q1, 0) = q1（0を読んでも偶奇変わらず） / δ(q1, 1) = q0（1を読んで偶数に戻る）

判定の手順（例: 入力 = 1101）

最終状態: q1 ← 受理状態ではない。「1101」の1の出現回数は3回（奇数）→ 拒否 ✗

入力 = 110 の場合: 1→q1, 1→q0, 0→q0。最終状態: q0 ★ → 受理 ✅（1が2回 = 偶数）

途中で q0（受理状態）を2回通過するが、最後が q0 なので受理 ✅。受理状態 = 「入力を全部読み終えたとき、そこにいるかどうか」だけで判定する。

間違えやすい点正しい理解
状態 = 「今どこにいるか」状態 = 「これまでの入力の抽象的な記憶」
受理状態を途中で通過 → 受理最後に受理状態にいる場合のみ受理
状態数は入力長に依存する状態数は有限で固定（入力長と無関係）
DFA はリアルタイムの動作を表すDFA は記憶更新規則を定めた数学的な仕様

確認問題

Q1. 形式言語において「言語」とは何か。

正解: 文字列の集合
言語 = 規則を満たす文字列の集合。例:「0で始まる文字列の集合」= {0, 00, 01, 010, …}

Q2. アルファベット Σ = {0, 1} のとき、正しい文字列はどれか。「abc」「011」「012」

正解: 011
アルファベットに含まれる記号（0と1）だけで構成されているのは「011」のみ。「abc」はaもbもcも Σ に含まれない。「012」の2も含まれない。

Q3. 正規表現 0*1 にマッチする文字列はどれか。「1」「10」「001」

正解: 1 と 001
0*1 = 「0が0回以上、その後に1が1回」。「1」は0が0回＋1 → ✅。「001」は0が2回＋1 → ✅。「10」は1の後に0 → ✗。

Q4. DFAの遷移関数 δ(q, a) = q′ の意味を説明せよ。

正解: 状態 q のときに文字 a を読むと状態 q′ に遷移する
δ は記憶の更新規則。入力文字1つに対して次の状態が1つに決まる（決定性）。

Q5. DFAで「入力 1101 を受理するか」を判定するにはどうするか。

正解: 開始状態から1文字ずつ遷移し、最後の状態が受理状態かどうかを確認する
途中で受理状態を通過しても関係ない。全文字を読み終えたときの状態だけで判定する。

Q6. 受理状態の説明として正しいのはどれか。①入力の途中でそこを通過すれば受理　②入力をすべて読み終えたときそこにいれば受理

正解: ②
受理状態は「最終判定のための印」。入力列を全部読み終えたとき、最終状態が受理状態かどうかだけで受理/拒否を決める。

Q7. 「1の出現回数が偶数」を判定するDFAに必要な状態数は最小いくつか。

正解: 2
「偶数」と「奇数」の2種類の記憶で十分。DFAは判定に必要な最小限の状態しか持たない。

Q8. 正規表現 ab+ にマッチする文字列はどれか。「a」「ab」「b」「abb」

正解: ab と abb
ab+ = 「a の後に b が1回以上」。「a」は b がない → ✗。「b」は a がない → ✗。「ab」= a + b×1 → ✅。「abb」= a + b×2 → ✅。

Q9. 正規表現 01|10 にマッチするのはどれか。「00」「01」「11」「10」

正解: 01 と 10
| は OR。01 または 10 にマッチする。「00」「11」はどちらにも含まれない。

Q10. 形式言語の「記号」についての正しい説明はどれか。①記号0は「ゼロ」という値を意味する　②記号はそれ自体に意味を持たない単なるマークである

正解: ②
形式言語では記号は意味を持たない「形」として扱う。意味はコンパイラや人間が規則を与えたときに初めて生まれる。
