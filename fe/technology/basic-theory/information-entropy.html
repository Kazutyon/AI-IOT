情報量とエントロピー

情報の「驚き」を数値化する −log₂P の仕組みと、情報源の不確実性を表すエントロピーをFE試験向けに整理します。

重要用語

用語意味試験ポイント
情報量 I(x)ある出来事の「驚きの大きさ」。I = −log₂ P(x)確率が小さいほど情報量大。単位はbit
bit情報量の単位。「はい/いいえ」で区別できる情報量コインの表裏 = 1 bit
エントロピー H情報源の平均情報量。H = −Σ Pᵢ log₂ Pᵢ期待値と同じ構造。不確実性の尺度
情報源情報を発生する源。コイン・サイコロ・文字列などエントロピー計算の対象
対数 log₂底が2の対数。情報量をbitで測る数学的基盤log₂(1/2) = −1 → −log₂(1/2) = 1
最大エントロピーすべての結果が均等な確率のとき最大。値は log₂n公平なサイコロ = log₂6 ≈ 2.58 bit
最小エントロピー1つの結果に確率が集中するとき最小。値は0完全に予測可能な情報源

情報量とは

情報とは「不確実性が減る量」です。起こりやすいことが起きても驚かない（情報量小）、めったに起きないことが起きると驚く（情報量大）という直感を数式で表します。

情報量の定義

I(x) = −log₂ P(x)

  P(x): 出来事xの確率
  底が2の対数: 情報をbitで測るため
  マイナス符号: 確率が小さいほど値が大きくなるよう正にする

  コイン表（確率 1/2） → I = −log₂(1/2) = 1 bit
  サイコロ出目（確率 1/6） → I = −log₂(1/6) ≈ 2.58 bit

情報量の性質

  • 確率が小さいほど情報量は大きい
  • 情報量は常に 0 以上
  • 独立な出来事は足し算（コイン2回 = 1 + 1 = 2 bit）
  • 確率が 1/10 になるごとに情報量は約 3.3 bit 増加

エントロピー（情報量の平均）

情報量が「1回の驚き」なら、エントロピーは「この情報源から情報を受け取るたびに、平均してどれだけ驚くか」を表します。期待値と同じ構造で計算します。

エントロピーの定義

H = −Σ Pᵢ log₂ Pᵢ
  = Σ Pᵢ × I(xᵢ)   ← 情報量の期待値

公平なコイン（表0.5, 裏0.5）:
  H = −0.5×log₂0.5 − 0.5×log₂0.5
    = −0.5×(−1) − 0.5×(−1)
    = 0.5 + 0.5 = 1 bit

偏ったコイン（表0.9, 裏0.1）:
  H = −0.9×log₂0.9 − 0.1×log₂0.1
    ≈ 0.137 + 0.332 ≈ 0.47 bit  ← 予測しやすいので小さい

定義−log₂ P(x)−Σ Pᵢ log₂ Pᵢ
対象1つの出来事情報源全体
意味その出来事の「驚き」平均的な「驚き」（不確実性）
単位bitbit
計算に使う確率1つ全結果の確率を総和
エントロピー意味実例
大不確実性が高い・予測しにくい公平なサイコロ（≈2.58 bit）
小不確実性が低い・予測しやすい偏ったコイン（≈0.47 bit）
0完全に予測可能毎回同じ結果が出る情報源

エントロピーとデータ圧縮

エントロピーは「データをどこまで圧縮できるかの理論的限界」を示します。ZIP・PNG・MP3 などの圧縮技術はすべてこの原理に基づきます。

エントロピー圧縮のしやすさ理由
大（高い不確実性）圧縮しにくいパターンがなく規則性が低い
小（低い不確実性）圧縮しやすい同じパターンが繰り返し出現する

間違えやすいポイント

  • log₂ の計算 — log₂(1/2) = −1 なので −log₂(1/2) = 1。「2の何乗か」で考える
  • 情報量とエントロピーの混同 — 情報量は1つの出来事、エントロピーは情報源全体の平均
  • エントロピー最大の条件 — 均等なとき最大（log₂n）。偏るほど小さくなる
  • エントロピー最小 = 0 — 1つの結果に確率が集中（完全予測可能）のとき
  • 単位は常にbit — 底が2の対数を使うため

確認問題

Q1. 公平なコインの表が出る情報量は何 bit か。

正解: 1 bit
I = −log₂(1/2) = −(−1) = 1 bit。「2の何乗か」で考えると 2¹ = 2 なので 1/2 → 1 bit。

Q2. サイコロの1の目が出る情報量は、コインの表裏よりどうなるか。

正解: 大きい
サイコロ: −log₂(1/6) ≈ 2.58 bit。コイン: 1 bit。確率 1/6 < 1/2 なので情報量は大きい。

Q3. 公平なコイン（表0.5, 裏0.5）のエントロピーは何 bit か。

正解: 1 bit
H = −0.5×log₂0.5 − 0.5×log₂0.5 = 0.5 + 0.5 = 1 bit。

Q4. 公平なサイコロ（6面）のエントロピーはいくつか。

正解: log₂6 ≈ 2.58 bit
結果が均等（各1/6）のときエントロピーは最大で log₂n。n=6 なので log₂6 ≈ 2.58 bit。

Q5. エントロピーが 0 になるのはどのような場合か。

正解: 1つの結果の確率が 1（完全に予測可能）
確率が1の結果しか出ない場合、不確実性はゼロ。−1×log₂1 = 0 bit。

Q6. 表0.9・裏0.1の偏ったコインのエントロピーは、公平なコインと比べてどうなるか。

正解: 小さい（約 0.47 bit）
偏っているほど予測しやすく、不確実性が低いためエントロピーは小さくなる。

Q7. エントロピーが大きいデータの特徴として正しいものはどれか。

正解: 圧縮しにくい
エントロピーが大きい = 不確実性が高い = パターンがない = 圧縮困難。ZIP などの圧縮効率が低い。

Q8. コインを2回独立に投げたとき、両方の結果の合計情報量はいくつか。

正解: 2 bit
独立な出来事の情報量は加算。1 bit + 1 bit = 2 bit。

Q9. 情報量 I(x) の式として正しいものはどれか。

正解: −log₂ P(x)
マイナス符号により確率が小さいほど正の値（情報量大）になる。底は2（単位: bit）。

Q10. 4つの結果が均等な確率（各1/4）の情報源のエントロピーはいくつか。

正解: 2 bit
均等なとき H = log₂n = log₂4 = 2 bit。または −4×(1/4×log₂(1/4)) = −(−2) = 2 bit。
