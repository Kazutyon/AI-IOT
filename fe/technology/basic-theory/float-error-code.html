固定小数点と浮動小数点

固定小数点数の仕組み

小数点の位置をあらかじめ決めた表現方式。例として16ビット中、上位8ビットを整数部・下位8ビットを小数部に割り当てる。

小数部のビット重みは左から 2⁻¹、2⁻²、2⁻³… と続く。

固定小数点 計算例（16ビット）

 00000001  10000000
 ────────  ────────
  整数部    小数部
    1     +  2⁻¹ = 0.5

→ 全体の値: 1 + 0.5 = 1.5

固定小数点 vs 浮動小数点 比較

項目固定小数点浮動小数点（IEEE 754 単精度）
ビット構成整数部＋小数部（任意）符号1＋指数8＋仮数23
小数点の位置形式で固定指数部で動的
表現範囲狭い広い（±約3.4×10³⁸）
計算速度速い（整数演算）やや遅い（専用FPU）
誤差ほぼなし必ず発生
主な用途組込み・制御・DSP科学計算・3DCG・一般アプリ

IEEE 754 単精度の構造

IEEE 754 は浮動小数点数の国際規格。単精度（32ビット）と倍精度（64ビット）が試験に出る。

領域ビット数役割
符号部1ビット0＝正、1＝負
指数部8ビットバイアス127付き。E＝E_stored − 127
仮数部23ビット正規化で先頭1省略（hidden bit）。実質24ビット精度

バイアス付き指数

指数部は「バイアス値127を加えた値」として保存する。これにより、負の指数（−127〜0）を符号なし整数で表現できる。

  • 指数部が 10000001₂ = 129 → 実際の指数 = 129 − 127 = 2
  • 指数部が 01111110₂ = 126 → 実際の指数 = 126 − 127 = −1

隠れた1（hidden bit）

正規化数では仮数部は常に 1.XXXXX… の形になる。先頭の "1" は必ず1なので省略できる。これにより、仮数部23ビットで実質24ビット分の精度を得る。

IEEE 754 単精度 解釈手順

[手順]
1. 先頭1ビット → 符号（0=正、1=負）
2. 次の8ビット → 指数部の保存値
3. バイアス127を引く → 実際の指数
4. 残り23ビット → 仮数部（先頭に "1." を付ける）
5. 値 = (−1)^符号 × 仮数 × 2^指数

[練習]
01000001001000000000000000000000
符号 = 0（正）
指数 = 10000010₂ = 130 → 130 − 127 = 3
仮数 = 1.0100…₂ = 1 + 0.25 = 1.25
値 = +1 × 1.25 × 2³ = 10.0

3大誤差

浮動小数点の誤差はバグではなく本質的な性質。仮数部が有限ビットである以上、誤差は避けられない。

誤差の種類発生条件具体例対策
丸め誤差値を有限ビットで表せない0.1（2進：循環小数）→ 0.1+0.2≠0.3ε比較・10進演算ライブラリ
桁落ち近い値同士の引き算1.234567−1.234566 → 有効桁が6桁→1桁式変形（有理化・因数分解）
情報落ち大きな数＋小さな数1.0×10²⁰+1.0 → +1が無視される小さい数から先に加算

丸め誤差のメカニズム

誤差の判定手順

計算式から誤差を分類する

[判定の手がかり]
  近い値の引き算      → 桁落ち
  大きな数 ＋ 小さな数 → 情報落ち
  小数を2進変換       → 丸め誤差
  無限級数を有限回で打切り → 打切り誤差

[例]
  1000000.0 − 999999.9 → 桁落ち
  1.0×10¹⁰ + 1.0      → 情報落ち
  0.1 + 0.2            → 丸め誤差

間違えやすいポイント

  • 丸め誤差はバグではない: 有限ビットで実数を表すことの本質的制約。
  • 誤差の分類を混同しない: 「近い値の引き算→桁落ち」「大+小→情報落ち」「小数2進変換→丸め誤差」
  • オーバーフロー vs アンダーフロー: オーバーは最大値超え（∞）、アンダーは最小正規化数未満（0）

文字コード

コンピュータは0/1のみ扱う。「どの文字にどの番号を割り当てるか」がルールが文字コードと文字集合の役割。

文字コード種類ビット数日本語英語特徴
ASCII文字コード7ビット（1B）×1B/文字英数字128文字のみ
JIS（シフトJIS）文字コード1〜2B2B1B日本独自。2バイト目に注意
Unicode文字集合最大21ビットU+3042などU+0041など保存形式ではない
UTF-8エンコード方式1〜4B3B1BASCII互換。Web標準

ASCII の主なコード値

  • 数字 '0'〜'9': 0x30〜0x39
  • 大文字 'A'〜'Z': 0x41〜0x5A
  • 小文字 'a'〜'z': 0x61〜0x7A

Unicode と UTF-8 の違い

Unicode は「文字集合（カタログ）」— 文字に番号（コードポイント）を割り当てるルール。保存形式ではない。

UTF-8 は「エンコード方式（保存形式）」— Unicodeコードポイントをバイト列に変換する方法。

バイト数先頭ビットパターン対象
1バイト0xxxxxxxASCII（0〜127）
2バイト110xxxxx 10xxxxxxラテン文字拡張など
3バイト1110xxxx 10xxxxxx 10xxxxxx日本語・中国語など
4バイト11110xxx 10xxxxxx 10xxxxxx 10xxxxxx絵文字など

間違えやすいポイント

  • Unicode ≠ UTF-8: Unicodeは「文字集合」、UTF-8は「エンコード方式」。混同しない。
  • ASCII互換の範囲: UTF-8でASCIIと互換なのは0〜127（1バイト文字）のみ。
  • シフトJISの注意: 2バイト目にASCIIと同じ範囲の値を使うため、バイト単位処理で文字化けを起こす。

確認問題

Q1. IEEE 754 単精度（32ビット）の符号部・指数部・仮数部のビット数はどれか。
A) 符号1・指数7・仮数24   B) 符号1・指数8・仮数23   C) 符号2・指数8・仮数22   D) 符号1・指数11・仮数20

答え: B) 符号1・指数8・仮数23

合計32ビット。倍精度は符号1・指数11・仮数52で合計64ビット。

Q2. 浮動小数点の正規化で仮数部の先頭ビットが省略される理由はどれか。
A) 省略しないと2バイトになるから   B) 正規化で先頭が常に1になるため   C) 先頭ビットは壊れやすいから   D) 省略しないとCPUが遅くなるから

答え: B) 正規化で先頭が常に1になるため

正規化すると仮数部は必ず 1.XXX の形になる。先頭の1は必ず1なので省略できる（hidden bit）。これで23ビットが実質24ビット精度になる。

Q3. Unicodeに関する説明として正しいものはどれか。
A) 1文字を常に2バイトで表現する   B) 世界中の文字に一意の番号を割り当てる文字集合である   C) 日本独自の文字コードである   D) 英数字のみを扱う7ビットの文字コードである

答え: B) 世界中の文字に一意の番号を割り当てる文字集合である

Unicodeは「文字集合（番号表）」であり保存形式ではない。UTF-8・UTF-16などがエンコード方式。Dは ASCII の説明。

Q4. IEEE 754 単精度でバイアス値（127）を指数部に加える理由はどれか。
A) 計算を速くするため   B) 負の指数を符号なし整数で表現するため   C) オーバーフローを防ぐため   D) 丸め誤差を減らすため

答え: B) 負の指数を符号なし整数で表現するため

指数が負の値（−127〜0）をとる場合でも、符号なし整数として保存するためにバイアス127を加える。読み出す時は格納値から127を引く。

Q5. 丸め誤差が発生する根本原因はどれか。
A) CPUの計算が遅いため   B) 仮数部が有限ビットのため   C) メモリが不足しているため   D) プログラムのバグのため

答え: B) 仮数部が有限ビットのため

仮数部のビット数が有限なので、無限小数は途中で打ち切らざるを得ない。これはバグではなく浮動小数点の本質的性質。

Q6. 1.234567 − 1.234566 で発生する主な誤差はどれか。
A) 丸め誤差   B) 桁落ち   C) 情報落ち   D) 打切り誤差

答え: B) 桁落ち

近い値同士の引き算で上位桁が相殺され、有効桁数が激減する現象。1.234567−1.234566=0.000001 で有効桁が6桁→1桁に。

Q7. 1.0×10²⁰ + 1.0 で発生する主な誤差はどれか。
A) 丸め誤差   B) 桁落ち   C) 情報落ち   D) オーバーフロー

答え: C) 情報落ち

指数的に大きな差があるため、＋1.0が仮数部から溢れて無視される。対策は小さい数から先に加算するか、Kahanの総和アルゴリズムを使う。

Q8. UTF-8の特徴として正しいものはどれか。
A) すべての文字を2バイトで表現する   B) ASCIIと互換性がない   C) 可変長で英数字1B・日本語3B・絵文字4B   D) 7ビット固定長の文字コードである

答え: C) 可変長で英数字1B・日本語3B・絵文字4B

UTF-8は可変長（1〜4バイト）でASCIIと完全互換。Aは誤り（UTF-16の誤解）。Dは ASCII の説明。

Q9. 固定小数点数と浮動小数点数の違いとして正しいものはどれか。
A) 浮動小数点のほうが計算が速い   B) 固定小数点のほうが表現範囲が広い   C) 浮動小数点は誤差が発生せず正確である   D) 固定小数点は小数点の位置が固定で範囲は狭いが誤差が起きにくい

答え: D) 固定小数点は小数点の位置が固定で範囲は狭いが誤差が起きにくい

固定小数点は整数演算に近く速いが範囲が狭い。浮動小数点は広範囲を扱えるが誤差が必ず発生する。
