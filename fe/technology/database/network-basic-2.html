1. ポート番号

ポート番号とは、同じIPの中で、どのサービスに通信を届けるかを決める番号。

ポート番号用途説明
80HTTP暗号化なしのWeb通信
443HTTPS暗号化ありのWeb通信
22SSHサーバー管理用の安全な接続
3306MySQLデータベース接続用（内部のみ公開）

ブラウザはURLのプロトコルによって自動でポートを選ぶ。http:// → 80、https:// → 443。

2. TCP（Webサーバーが通信路を作る仕組み）

WebサーバーはHTTP/HTTPSを使って通信するが、HTTP/HTTPSはTCPの上で動くプロトコル。つまり、WebサーバーはTCPが作った通信路の上でしか動けない。

層役割Webサーバーとの関係 TCP通信路を作る（接続の確立・維持・終了）Webサーバーが通信できる状態を作る HTTP/HTTPSリクエストとレスポンスの中身Webサーバーが処理するデータそのもの

つまり：TCPが「道」を作り、HTTPが「荷物」を運ぶ、Webサーバーは荷物を受け取り返す。

工程TCPの動作Webサーバー側の意味
① SYN接続要求を送信「接続要求が来た」と認識
② SYN/ACK接続を受け入れ応答「接続を許可した」状態
③ ACK応答を確認TCP接続が確立（通信路ができる）
④ HTTP通信TCPの上でデータを送受信リクエストを受け取りレスポンスを返す
⑤ FIN切断要求接続終了を開始

3. ファイアウォールの重要性

ファイアウォール（FW）は、Webサーバーがインターネットからアクセスされる工程の中で、「どの通信を通し、どの通信を拒否するか」を決める境界装置。WebサーバーはLAN内のプライベートIPで動作しているため、外部からの通信は必ず NAT → ファイアウォール → Webサーバー の順に通過する。

ポート用途必要性
80HTTP暗号化なしのWebアクセス
443HTTPS暗号化されたWebアクセス（実質必須）

逆に通してはいけない代表的な通信：

ポート用途なぜ拒否するか
22SSH外部に公開すると不正ログイン攻撃を受ける
3306MySQLDBを外部公開すると情報漏洩リスク
21FTPパスワード盗聴の危険

Webサーバーは80/443以外は基本的に外部へ公開しない。

4. ファイアウォールが関わる工程

Webサーバーに通信が届くまでの工程：

  1. DNS：ドメイン名 → グローバルIP
  2. NAT：グローバルIP → プライベートIP
  3. ファイアウォール：通信を許可/拒否
  4. Webサーバー：リクエストを処理

ファイアウォールはNATの後に通信をチェックする。

間違えやすいポイント

  • ポート番号とIPアドレスの混同：IPは「どのコンピュータか」、ポートは「そのコンピュータのどのサービスか」を識別する。
  • TCPとHTTPの関係の誤解：TCPが通信路を作り、その上でHTTPが動作する。WebサーバーはTCP接続の上でHTTPを処理する。
  • 3ウェイハンドシェイクの順序の混乱：SYN（要求）→ SYN/ACK（応答）→ ACK（確認）の順序を覚える。
  • ファイアウォールで通すべきポートの誤り：Webサーバー公開では80と443のみを通し、22や3306は通さない。
  • ファイアウォールとNATの順序：正しい順序は「NAT → ファイアウォール → Webサーバー」。

確認問題

Q1. HTTPSが使用するポート番号はどれか。

ア）20　イ）80　ウ）443　エ）8080

答えを見る ▼

答え: ウ — HTTPSは443番ポートを使用する。80番ポートはHTTPが使用する。

Q2. SSH（サーバー管理）が使用するポート番号はどれか。

ア）22　イ）80　ウ）443　エ）3306

答えを見る ▼

答え: ア — SSHは22番ポートを使用する。Webサーバー公開時は通常外部に公開しない。

Q3. TCPの3ウェイハンドシェイクの正しい順序はどれか。

ア）ACK → SYN → SYN/ACK　イ）SYN → ACK → SYN/ACK　ウ）SYN → SYN/ACK → ACK　エ）SYN/ACK → SYN → ACK

答えを見る ▼

答え: ウ — 3ウェイハンドシェイクは「SYN（接続要求）→ SYN/ACK（受理）→ ACK（確認）」の順序で行われる。

Q4. TCPの役割として正しいものはどれか。

ア）リクエストとレスポンスの中身を決める　イ）通信路を作る（接続の確立・維持・終了）　ウ）データを暗号化する　エ）ドメイン名をIPアドレスに変換する

答えを見る ▼

答え: イ — TCPは通信路を作るプロトコル。HTTP/HTTPSはTCPの上で動く。

Q5. Webサーバーをインターネットに公開する際、ファイアウォールで許可すべきポートはどれか。

ア）22と80　イ）80と443　ウ）443と3306　エ）22と3306

答えを見る ▼

答え: イ — Webサーバー公開ではHTTP（80）とHTTPS（443）を通す。SSH（22）やMySQL（3306）は通さない。

Q6. ファイアウォールの役割として正しいものはどれか。

ア）ドメイン名をIPアドレスに変換する　イ）プライベートIPとグローバルIPを変換する　ウ）どの通信を通し、どの通信を拒否するか決める　エ）通信内容を暗号化する

答えを見る ▼

答え: ウ — ファイアウォールはポート番号・IPアドレス・プロトコルを基準に通信の許可/拒否を判断する。

Q7. Webサーバーに通信が届くまでの工程として正しい順序はどれか。

ア）DNS → NAT → ファイアウォール → Webサーバー　イ）NAT → DNS → ファイアウォール → Webサーバー　ウ）ファイアウォール → DNS → NAT → Webサーバー　エ）DNS → ファイアウォール → NAT → Webサーバー

答えを見る ▼

答え: ア — 正しい順序は「DNS（名前解決）→ NAT（IP変換）→ ファイアウォール（通信許可）→ Webサーバー」。

Q8. 「1つのサーバーに複数のサービスが動作している」とき、どのサービスに通信を届けるかを区別するために使われるものはどれか。

ア）MACアドレス　イ）ポート番号　ウ）サブネットマスク　エ）デフォルトゲートウェイ

答えを見る ▼

答え: イ — ポート番号は同じIPの中で、どのサービスに通信を届けるかを決める番号である。

Q9. TCP接続の切断を要求するときに送信されるフラグはどれか。

ア）SYN　イ）ACK　ウ）FIN　エ）RST

答えを見る ▼

答え: ウ — FINフラグはTCP接続の切断を要求するときに送信される。

Q10. Webサーバー公開において、外部に公開してはいけないポートの例として適切なものはどれか。

ア）80　イ）443　ウ）22　エ）8080

答えを見る ▼

答え: ウ — SSHの22番ポートは外部に公開すると不正ログイン攻撃の対象となるため、公開してはいけない。
