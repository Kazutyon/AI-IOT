1. ステータスコード

ステータスコードは、Webサーバーが「リクエストをどう処理したか」をブラウザに伝えるための結果コード。HTTPレスポンスの1行目に必ず含まれる。

代表的なステータスコード

コード意味発生する状況ブラウザの動作
200 OK成功リクエストを正常に処理できたそのまま表示する
301 Moved Permanently恒久的リダイレクトURL変更・常時HTTPS化新しいURLをブックマーク
302 Found一時的リダイレクトログイン後の遷移毎回リクエスト
404 Not Found存在しないURLの間違い・削除済み「見つかりません」と表示
500 Internal Server Errorサーバー内部エラーバグ・例外・DBエラー再試行しても改善しない

リダイレクトの種類

コードメソッドの扱い用途
301GETに変わる可能性ありURL変更・常時HTTPS化
308メソッドを変えないAPIの恒久移動
302GETに変わる可能性ありログイン後の遷移
307メソッドを変えないPOSTの再送防止

2. HTTPS（Webサーバーが安全に通信するための仕組み）

HTTPSとは、HTTPの通信内容をTLSによって暗号化し、Webサーバーとブラウザ間の通信を安全にする仕組み。

理由Webサーバー側の意味
通信内容の盗聴防止パスワード・Cookie・個人情報を守る
通信内容の改ざん防止中間者攻撃（MITM）を防ぐ
ブラウザの警告回避Chrome/Edge/Safariが警告を出さない
CookieのSecure属性に必須セッション管理の安全性向上
SEO（検索順位）にも影響GoogleがHTTPSを推奨

3. SSL証明書

SSL証明書は、TLSハンドシェイクの工程でWebサーバーがブラウザに提示する「身分証明書」。ブラウザは証明書を見て、このサーバーは本物か、信頼できるCAが発行しているか、有効期限内かを確認する。

4. HTTPS導入の流れ

工程作業内容
① 証明書を入手する認証局（CA）からSSL証明書を発行してもらう。Let's Encrypt（無料）や商用CAがある。
② CSRを作成するドメイン名・組織名・公開鍵を含む申請書を作成しCAに提出
③ Webサーバーに証明書を組み込む証明書（CRT）・秘密鍵（KEY）・中間証明書を443番ポートのHTTPS設定に紐づける
④ 443番ポートを公開するファイアウォールで443番を許可、NATでグローバルIP:443 → Webサーバー:443に転送
⑤ HTTP→HTTPSにリダイレクト80番で受け取り「HTTPSに移動してください」（301/308）を返す

5. HTTPからHTTPSへの誘導

ユーザーが http://example.com にアクセスする場合がある。そのためWebサーバーは：

  1. 80番で受け取る
  2. 「HTTPSに移動してください」（301/308）を返す
  3. ブラウザが自動で https://example.com に移動する

間違えやすいポイント

  • ステータスコードの分類の混同：4xxはクライアント側の問題（URL間違い・権限不足）、5xxはサーバー側の問題（バグ・障害）。原因を区別することが重要。
  • 301と302の違い：301は恒久的な移動（ブラウザは新しいURLを覚える）、302は一時的な移動（毎回元のURLにアクセス）。
  • HTTPS＝完全な安全ではない：HTTPSは通信路の暗号化であり、サーバー内部の脆弱性やアプリケーションのバグは防げない。
  • SSL証明書と暗号化の関係の誤解：証明書は「サーバーの身元証明」が主目的。暗号化自体は証明書に含まれる公開鍵を使って行う。
  • ポート番号の混同：HTTPは80、HTTPSは443。HTTPSサイトにhttp://でアクセスするとポート80に接続しようとしてエラーになる。

確認問題

Q1. HTTPステータスコードの分類で、「サーバーエラー」を示すものはどれか。

ア）2xx　イ）3xx　ウ）4xx　エ）5xx

答えを見る ▼

答え: エ — 5xx（500番台）はサーバーエラーを示す。4xxはクライアントエラー、2xxは成功、3xxはリダイレクト。

Q2. リクエストされたリソースが存在しない場合に返されるステータスコードはどれか。

ア）200 OK　イ）302 Found　ウ）404 Not Found　エ）500 Internal Server Error

答えを見る ▼

答え: ウ — 404 Not FoundはリクエストされたURLに対応するリソースがサーバー上に存在しない場合に返される。

Q3. 恒久的なリダイレクト（URLが永久に移動した）を示すステータスコードはどれか。

ア）200 OK　イ）301 Moved Permanently　ウ）302 Found　エ）307 Temporary Redirect

答えを見る ▼

答え: イ — 301は恒久的なリダイレクト。302/307は一時的なリダイレクト。

Q4. HTTPSが必要な理由として適切でないものはどれか。

ア）通信内容の盗聴防止　イ）通信内容の改ざん防止　ウ）サーバー内部のバグ防止　エ）サーバーの正当性証明

答えを見る ▼

答え: ウ — HTTPSは通信路の暗号化であり、サーバー内部のバグやアプリケーションの脆弱性は防げない。

Q5. HTTPSが使用するポート番号はどれか。

ア）80　イ）443　ウ）8080　エ）3306

答えを見る ▼

答え: イ — HTTPSは443番ポートを使用する。80番はHTTP、3306はMySQL。

Q6. SSL証明書の主な役割はどれか。

ア）通信内容を圧縮する　イ）サーバーの正当性を証明する　ウ）データベースに接続する　エ）HTMLをキャッシュする

答えを見る ▼

答え: イ — SSL証明書は「このサーバーは本物のexample.comです」と証明する電子証明書。

Q7. Webサーバー内部でバグや例外が発生した場合に返されるステータスコードはどれか。

ア）403 Forbidden　イ）404 Not Found　ウ）500 Internal Server Error　エ）503 Service Unavailable

答えを見る ▼

答え: ウ — 500 Internal Server Errorはサーバー内部で予期せぬエラーが発生した場合に返される。

Q8. HTTPからHTTPSへのリダイレクトを設定する際、Webサーバーが返すべきステータスコードはどれか。

ア）200 OK　イ）301 Moved Permanently　ウ）404 Not Found　エ）500 Internal Server Error

答えを見る ▼

答え: イ — HTTPSへの移行は恒久的な変更であるため、301 Moved Permanentlyを使用するのが一般的。

Q9. TLSハンドシェイクの説明として正しいものはどれか。

ア）HTTPリクエストを送信する手順　イ）鍵交換と証明書提示を行う手順　ウ）HTMLを解析する手順　エ）データベースに接続する手順

答えを見る ▼

答え: イ — TLSハンドシェイクはHTTPS通信開始時の鍵交換と証明書提示の手順。この後で暗号化されたHTTP通信が始まる。

Q10. 認証局（CA）の役割として正しいものはどれか。

ア）Webサーバーの負荷を分散する　イ）SSL証明書を発行する　ウ）HTMLをキャッシュする　エ）DNSを管理する

答えを見る ▼

答え: イ — 認証局（Certificate Authority）はSSL証明書を発行する信頼できる第三者機関。
