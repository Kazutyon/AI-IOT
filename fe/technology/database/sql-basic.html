1. SQLとは

SQL（Structured Query Language）は、RDBを操作するための標準言語である。Webアプリケーションは、ユーザー登録・ログイン・商品一覧・注文処理など、ほぼすべての機能でSQLを実行してデータを扱う。

SELECT（検索）

SELECTはデータを取得するためのSQL。

INSERT（登録）

INSERTは新しいデータを追加するためのSQL。

UPDATE（更新）

UPDATEは既存データを変更するためのSQL。

DELETE（削除）

DELETEはデータを削除するためのSQL。

CRUDとWebアプリの関係

SQLの4操作は、Webアプリの基本機能と完全に対応している。

SQL意味Webアプリの機能例
SELECT検索一覧表示 / 詳細表示
INSERT登録新規登録 / 投稿 / 注文作成
UPDATE更新プロフィール編集 / 在庫更新
DELETE削除データ削除 / 論理削除

論理削除とは

実務では、データを完全に削除する「物理削除」ではなく、「is_deleted = 1」などのフラグで削除を管理する「論理削除」を使うことが多い。理由：誤削除からの復旧が容易、削除履歴を残せる、参照整合性を保ちやすい。

間違えやすいポイント

  • UPDATE/DELETEでWHERE句を付け忘れる：WHEREがないと全件更新/全件削除されてしまう。重大事故につながる。
  • SELECT * の多用：必要なカラムだけを指定しないと、不要なデータまで取得して性能が低下する。
  • 文字列の引用符忘れ：SQLでは文字列はシングルクォート（'）で囲む。数字はそのまま。
  • INSERTのカラム順とVALUESの順序不一致：カラム名を省略するとテーブル定義の順序に依存するため、明示的に指定するのが安全。

Q1. データを取得するためのSQLはどれか。

ア）INSERT　イ）SELECT　ウ）UPDATE　エ）DELETE

答えを見る ▼

答え: イ — SELECTはデータを取得するためのSQL。一覧表示や詳細表示で使用する。

Q2. 新しいデータを追加するためのSQLはどれか。

ア）INSERT　イ）SELECT　ウ）UPDATE　エ）DELETE

答えを見る ▼

答え: ア — INSERTは新しいデータを追加するためのSQL。新規登録や投稿で使用する。

Q3. UPDATE文でWHERE句を付け忘れるとどうなるか。

ア）エラーになる　イ）何も更新されない　ウ）全件更新される　エ）最初の1件だけ更新される

答えを見る ▼

答え: ウ — UPDATE文でWHERE句を付け忘れると、テーブルの全レコードが更新される。重大な事故につながる。

Q4. DELETE文でWHERE句を付け忘れるとどうなるか。

ア）エラーになる　イ）何も削除されない　ウ）全件削除される　エ）最初の1件だけ削除される

答えを見る ▼

答え: ウ — DELETE文でWHERE句を付け忘れると、テーブルの全レコードが削除される。重大な事故につながる。

Q5. Webアプリの「プロフィール編集」機能に対応するSQLはどれか。

ア）INSERT　イ）SELECT　ウ）UPDATE　エ）DELETE

答えを見る ▼

答え: ウ — プロフィール編集は既存データを変更するため、UPDATEを使用する。

Q6. CRUDの「R」が指すSQLはどれか。

ア）INSERT　イ）SELECT　ウ）UPDATE　エ）DELETE

答えを見る ▼

答え: イ — CRUDはCreate/Read/Update/Deleteの略。R（Read）はSELECTに対応する。

Q7. 実務で物理削除ではなく論理削除が使われる理由として適切でないものはどれか。

ア）誤削除からの復旧が容易　イ）削除履歴を残せる　ウ）ストレージ容量を節約できる　エ）参照整合性を保ちやすい

答えを見る ▼

答え: ウ — 論理削除はデータを残すため、物理削除よりもストレージ容量を消費する。容量節約は目的ではない。

Q8. SELECT文で「すべてのカラム」を意味する記号はどれか。

ア）?　イ）%　ウ）*　エ）_

答えを見る ▼

答え: ウ — アスタリスク（*）は「すべてのカラム」を意味する。例：SELECT * FROM users;

Q9. Webアプリの「商品検索」機能に対応するSQLはどれか。

ア）INSERT　イ）SELECT　ウ）UPDATE　エ）DELETE

答えを見る ▼

答え: イ — 商品検索はデータを取得するため、SELECTを使用する。

Q10. 次のSQL文のうち、正しいものはどれか。

ア）SELECT name, email FROM users;　イ）INSERT INTO users VALUES ('山田太郎');　ウ）UPDATE users name = '花子';　エ）DELETE users WHERE id = 1;

答えを見る ▼

答え: ア — SELECT name, email FROM users; は正しい構文。ウはSET句が不足、エはDELETE FROMが正しい。
