1. Webサーバーソフトウェアの全体像

Webページは、ブラウザ → Webサーバー → アプリケーションという流れで処理される。実際にリクエストを受け取り、ファイルを返したり、アプリケーションに処理を渡したりしている「本体」がWebサーバーソフトウェア（Apache / Nginx）である。

2. Webサーバーソフトウェアが提供する主な機能

Apache / Nginxは、次のような機能を設定できる。

機能説明
静的ファイルの配信HTML、CSS、JS、画像などをクライアントに返す
アプリケーションへの処理受け渡しPHP（php-fpm）、Node.js、Pythonなどに処理を渡す
ログの記録access.log（アクセス記録）とerror.log（エラー記録）
エラーページの制御404や500のエラーページをカスタマイズ
仮想ホスト1台のサーバーで複数のWebサイトを公開
ファイル配置のルールDocumentRootやpublic_htmlの設定
HTTPS（TLS）の設定那样SSL証明書の設定、暗号化通信の開始
性能改善Keep-Alive、gzip圧縮、キャッシュ制御
アクセス制御那样IP制限、Basic認証

3. Apache / Nginxが扱う「対象」の一覧

Webサーバーソフトウェアは、次のような「対象」を設定することで動作を制御する。

対象説明
URLパスどのURLに対してどの処理を行うか
ファイルパスどのディレクトリを公開するか
ホスト名（ドメイン）example.com / api.example.com など
ポート番号80 / 443
ログファイル記録場所・形式
エラーページ404.html / 500.html
プロセス/ワーカー数同時処理数（性能に関係）

4. Apache / Nginxの設定でできること

① URLとファイルの対応付け（DocumentRoot）

/ にアクセスされたら /var/www/html/index.html を返す、/img/ は /var/www/html/img/ を返す、といったURL → ファイルの対応を設定する。

② 複数サイトの公開（仮想ホスト）

1台のサーバーで、example.com、shop.example.com、test.example.comのように複数のサイトを同時に公開できる。

③ エラーページの制御（404 / 500）

ファイルがない → 404.html、アプリがエラー → 500.htmlといった独自のエラーページを設定できる。

④ ログの記録と分析（access.log / error.log）

どのURLにアクセスされたか、ステータスコード、レスポンス時間、エラー内容などを記録し、運用に役立てる。

⑤ アプリケーションとの連携

PHP（php-fpm）、Node.js、Python（WSGI）など、動的ページを生成するアプリケーションに処理を渡す。

⑥ 性能改善（Keep-Alive / gzip / キャッシュ）

  • 接続を維持して高速化（Keep-Alive）
  • HTML/CSS/JSを圧縮（gzip）
  • キャッシュ制御（Expires / Cache-Control）
  • CDNと連携して高速化

⑦ HTTPS（TLS）の設定

  • 証明書の設定
  • 暗号化通信の開始
  • HTTP → HTTPS のリダイレクト

間違えやすいポイント

  • WebサーバーソフトウェアとOSの混同：Apache/Nginxはソフトウェアであり、OS（Windows/Linux）とは別物。
  • 静的ファイル配信と動的処理の違い：静的ファイルはWebサーバー自身が返し、動的処理はアプリケーションに渡す。
  • 設定ファイルの重要性：Webサーバーの動作はすべて設定ファイルで制御される。GUIで設定するものではない。
  • 仮想ホストの誤解：1台のサーバーで複数サイトを公開するにはIPアドレスが複数必要と思われがちだが、Hostヘッダを使えば1つのIPで可能。

確認問題

Q1. Webサーバーソフトウェアの代表的な例として正しい組み合わせはどれか。

ア）Chrome / Safari　イ）Apache / Nginx　ウ）Windows / Linux　エ）MySQL / PostgreSQL

答えを見る ▼

答え: イ — ApacheとNginxは代表的なWebサーバーソフトウェア。Chrome/Safariはブラウザ、Windows/LinuxはOS、MySQL/PostgreSQLはDB。

Q2. Webサーバーソフトウェアが提供する機能として適切でないものはどれか。

ア）静的ファイルの配信　イ）アプリケーションへの処理受け渡し　ウ）データベースの管理　エ）ログの記録

答えを見る ▼

答え: ウ — データベースの管理はDBMSの役割。WebサーバーはDBを直接管理しない。

Q3. 1台のサーバーで複数のWebサイトを公開する仕組みを何と呼ぶか。

ア）ロードバランシング　イ）仮想ホスト　ウ）リバースプロキシ　エ）キャッシュ

答えを見る ▼

答え: イ — 仮想ホスト（Virtual Host / Server Block）は、1台のサーバーで複数のWebサイトを公開する仕組み。

Q4. Webサーバーのログとして正しい組み合わせはどれか。

ア）system.log / debug.log　イ）access.log / error.log　ウ）security.log / audit.log　エ）query.log / slow.log

答えを見る ▼

答え: イ — Webサーバーの主なログはaccess.log（アクセス記録）とerror.log（エラー記録）の2種類。

Q5. Webサーバーが静的ファイルを返す際の処理として正しいものはどれか。

ア）アプリケーションでHTMLを生成する　イ）データベースからデータを取得する　ウ）ファイルシステムからファイルを読み取って返す　エ）リクエストを別のサーバーに転送する

答えを見る ▼

答え: ウ — 静的ファイルはファイルシステムに保存されており、Webサーバーはそれを読み取ってそのまま返す。

Q6. Webサーバーソフトウェアが動的処理をアプリケーションに渡す方式として使われるものはどれか。

ア）DNS　イ）NAT　ウ）FastCGI　エ）DHCP

答えを見る ▼

答え: ウ — FastCGI（php-fpmなど）はWebサーバーがアプリケーションに処理を渡す方式。

Q7. Webサーバーソフトウェアの設定ファイルで制御できる項目として適切でないものはどれか。

ア）DocumentRootの設定　イ）エラーページのカスタマイズ　ウ）CPUのクロック周波数　エ）HTTPSの設定

答えを見る ▼

答え: ウ — CPUのクロック周波数はハードウェアの設定であり、Webサーバーの設定ファイルでは制御できない。

Q8. Webサーバーがレスポンスをgzip圧縮する目的はどれか。

ア）セキュリティの向上　イ）転送量の削減と表示速度の向上　ウ）データベース接続の高速化　エ）CPU負荷の軽減

答えを見る ▼

答え: イ — gzip圧縮によりHTML/CSS/JSなどのテキストデータを圧縮し、転送量を減らして表示速度を向上させる。

Q9. Webサーバーにアクセスがあった記録を保存するログファイルはどれか。

ア）error.log　イ）access.log　ウ）system.log　エ）debug.log

答えを見る ▼

答え: イ — access.logはすべてのリクエストを記録する。error.logはエラーを記録する。

Q10. WebサーバーソフトウェアがHTTPSを提供するために必要なものはどれか。

ア）DNSレコード　イ）SSL証明書　ウ）ロードバランサー　エ）ファイアウォール

答えを見る ▼

答え: イ — HTTPSを提供するにはSSL証明書が必要。証明書をWebサーバーに設定し、443番ポートで待ち受ける。
