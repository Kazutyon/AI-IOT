1. OS（オペレーティングシステム）とは何か

OS（Operating System）は、コンピュータのハードウェアとアプリケーションの間に入り、両者を仲介する基盤ソフトウェア。Webサーバーを理解するためには、OSがどのように「ファイル」「プロセス」「ネットワーク」「権限」を管理しているかを概念として正確に理解する必要がある。

管理領域説明Webサーバーとの関係
ファイルHTML、画像、ログなどのデータを保存する場所WebサーバーはOSのファイル管理機能を使って静的ファイルを読み取る
プロセス動いているプログラムWebサーバー（Apache/Nginx）もアプリケーションもOS上で動く
ネットワークネットワークインターフェースの管理WebサーバーはOSを通じて通信を行う
権限アクセス制御Webサーバーが読み取れるファイルはOSの権限設定で決まる

2. サーバーOSの特徴

WebサーバーはWindowsでもLinuxでも動作するが、世界のWebサーバーの大半はLinux上で動いている。

理由説明
常時稼働に強い安定性が高く、長時間の連続動作に適している
不要な機能が少なく軽量サーバー用途に必要な機能に絞られている
サーバー用途に最適化されている大量アクセス処理やネットワーク性能に優れる
権限管理が明確セキュリティ面での管理が容易
WebサーバーがLinuxを前提に設計されているApache/NginxはLinux環境での動作を想定している

3. Linuxの種類（ディストリビューション）

Linuxは1つのOSではなく、共通のLinuxカーネルをもとに、さまざまな組織が独自に構成した「種類」が存在する。これをディストリビューション（Distribution）と呼ぶ。

ディストリビューション特徴
Ubuntu Server世界的に利用者が多い。設定情報が豊富。初心者向け。
Debian安定性が高い。Ubuntuの元になっている。
CentOS（終了）企業サーバーで長く使われた。現在は後継へ移行。
Rocky Linux / AlmaLinuxCentOSの後継。企業サーバーで採用が増えている。
Amazon LinuxAWS専用。クラウドでよく使われる。

レンタルサーバーで使われるLinux：多くのレンタルサーバーはCentOS系（現在はRocky/AlmaLinux）をベースにしている。理由は安定性が高く、Webサーバー用途に最適化されており、管理ツール（cPanel、Plesk）が対応しているため。

4. 開発環境を構築するためのパッケージ

Webサーバーの学習では、本番サーバー（Linux）とは別に、ローカルPC上で動作する開発環境を用意することが多い。開発環境では、Webサーバー・データベース・PHPなどをまとめてインストールできるパッケージがよく使われる。

パッケージ名対応OS内容特徴
XAMPPWindows / macOS / LinuxApache / MySQL / PHP / phpMyAdmin最も有名。学習用として広く使われる。
MAMPmacOS / WindowsApache / MySQL / PHPMacユーザーに人気。UIが使いやすい。
WAMPWindowsApache / MySQL / PHPWindows専用。設定が簡単。

開発環境パッケージを使う理由：本番サーバーと同じ構成をローカルで再現できる、ApacheやMySQLを個別にインストールする必要がない、設定が簡単で初心者でも扱いやすい。

5. 本番環境（Linux）との違い

項目開発環境パッケージ本番環境（Linux）
目的学習・開発公開・運用
OSWindows / macOSLinux
インストール一括個別（Apache, PHP, MySQL）
設定GUI中心設定ファイル中心
安定性低い高い
セキュリティ弱い強い

間違えやすいポイント

  • 「OS＝Windows」という思い込み：Webサーバーの大半はLinuxで動作する。試験ではLinuxを前提とした出題が多い。
  • ディストリビューションの混同：「Linux」という1つのOSがあるわけではなく、複数のディストリビューションが存在することを理解する必要がある。
  • 開発環境と本番環境の同一視：XAMPPなどのパッケージは学習用であり、本番環境はLinuxで設定ファイル中心の運用になる。
  • OSの管理領域のイメージ不足：「ファイル・プロセス・ネットワーク・権限」の4つをセットで覚えること。

確認問題

Q1. OS（オペレーティングシステム）が管理するものとして、適切でないものはどれか。

ア）ファイル　イ）プロセス　ウ）ブラウザ　エ）権限

答えを見る ▼

答え: ウ — OSが管理する4つの領域は「ファイル」「プロセス」「ネットワーク」「権限」である。ブラウザはOS上で動作するアプリケーション。

Q2. Webサーバーの大半がLinux上で動作する理由として、適切でないものはどれか。

ア）常時稼働に強い　イ）グラフィカルなデスクトップ環境が標準装備されている　ウ）権限管理が明確　エ）WebサーバーがLinuxを前提に設計されている

答えを見る ▼

答え: イ — LinuxサーバーはGUIデスクトップ環境が標準ではなく、不要な機能を省いて軽量に動作することが特徴である。

Q3. Linuxディストリビューションの説明として正しいものはどれか。

ア）Linuxは1つのOSであり、ディストリビューションは存在しない　イ）共通のLinuxカーネルをもとに、さまざまな組織が独自に構成した種類　ウ）ディストリビューションはWindows専用のOSである　エ）すべてのディストリビューションは商用ライセンスが必要である

答えを見る ▼

答え: イ — Linuxはオープンソースであり、誰でも自由に改良・再配布できるため、目的に応じて多くのディストリビューションが作られている。

Q4. 次のLinuxディストリビューションのうち、CentOSの後継として企業サーバーで採用が増えているものはどれか。

ア）Ubuntu Server　イ）Debian　ウ）Rocky Linux / AlmaLinux　エ）Amazon Linux

答えを見る ▼

答え: ウ — Rocky LinuxとAlmaLinuxはCentOSの後継ディストリビューションであり、企業サーバーでの採用が増えている。

Q5. 開発環境パッケージ（XAMPP/MAMP/WAMP）の特徴として適切なものはどれか。

ア）本番環境としても安定して使える　イ）Webサーバー・DB・PHPなどを一括でインストールできる　ウ）Linuxでしか動作しない　エ）設定はすべて設定ファイルで行う必要がある

答えを見る ▼

答え: イ — 開発環境パッケージはWebサーバー・DB・PHPなどを一括でインストールでき、初心者でも簡単にWebアプリの動作確認ができる。

Q6. 本番環境（Linux）と開発環境パッケージの違いとして正しいものはどれか。

ア）本番環境はGUI中心の設定、開発環境は設定ファイル中心　イ）本番環境は安定性が高い、開発環境はセキュリティが強い　ウ）本番環境は設定ファイル中心、開発環境はGUI中心の設定が多い　エ）本番環境は学習目的、開発環境は公開・運用が目的

答えを見る ▼

答え: ウ — 本番環境（Linux）は設定ファイル中心の設定を行い、開発環境パッケージはGUI中心の設定が多い。

Q7. OSの「プロセス」管理についての説明として正しいものはどれか。

ア）HTMLや画像を保存する場所を管理する　イ）WebサーバーもアプリケーションもOS上でプロセスとして動作する　ウ）ネットワーク通信の経路を管理する　エ）ユーザーのアクセス権限を管理する

答えを見る ▼

答え: イ — プロセスは「動いているプログラム」のことで、WebサーバーやアプリケーションもOS上でプロセスとして動作する。

Q8. レンタルサーバーでCentOS系（Rocky/AlmaLinux）がよく使われる理由として適切でないものはどれか。

ア）安定性が高い　イ）Webサーバー用途に最適化されている　ウ）ゲーム用途に特化している　エ）管理ツール（cPanel、Plesk）が対応している

答えを見る ▼

答え: ウ — ゲーム用途はレンタルサーバーの主要な用途ではなく、Webサーバーとしての安定性や管理ツールの対応が採用理由である。

Q9. OSの「権限（アクセス制御）」の説明として正しいものはどれか。

ア）どのユーザーがどのファイルを読めるかを決める仕組み　イ）ネットワーク通信の速度を制限する仕組み　ウ）CPUの処理時間を割り当てる仕組み　エ）メモリの使用量を制限する仕組み

答えを見る ▼

答え: ア — 権限（アクセス制御）は、どのユーザーがどのファイルを読めるか、Webサーバーが読み取れるファイルはOSの権限設定で決まる。

Q10. Ubuntu Serverの特徴として適切なものはどれか。

ア）AWS専用のLinuxディストリビューション　イ）安定性が非常に高く、Ubuntuの元になっている　ウ）世界的に利用者が多く、初心者向け　エ）CentOSの後継ディストリビューション

答えを見る ▼

答え: ウ — Ubuntu Serverは世界的に利用者が多く、設定情報が豊富で初心者向けのLinuxディストリビューションである。
