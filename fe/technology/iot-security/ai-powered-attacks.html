1. なぜAIが攻撃に使われるのか（背景）

攻撃者がAIを悪用する側面も持つ。攻撃者がAIを使うことで、攻撃の質（精度）と量（自動化）が大きく変化している。

攻撃者がAIを利用する理由は明確である：

  • 大量生成ができる（メール・画像・音声など）
  • 人間では気づけない精度で偽装できる
  • ターゲットに合わせて内容を最適化できる
  • 攻撃の自動化・高速化が可能

つまりAIは、攻撃者にとって「コストを下げ、成功率を上げる道具」になっている。

2. AIが使われる主な攻撃手法

◆ ディープフェイク（AIを使った音声・映像の偽造）

手法の本質：AIを使って本人そっくりの音声・映像を「生成・合成」し、なりすます攻撃。「本物のように見える／聞こえる偽物」を作り、送金指示・詐欺・信用失墜に使う。

項目内容
主な用途送金指示・なりすまし詐欺・信用失墜・偽情報拡散
危険性人間が見ても聞いても本物と区別できない。「映像・音声だから信頼できる」という前提が崩壊
リモートワーク時代のリスク対面確認が減ったことで攻撃成功率が高い

ディープフェイクの実例

  • 香港企業の「偽オンライン会議」事件（2023年）：攻撃者が複数社員の顔と声をAIで偽造。Zoom会議に偽物の同僚が参加し、財務担当者に送金指示。約2億円を送金。
  • 英国企業の「CEOなりすまし音声」事件（2019年）：AIでCEOの声を模倣し、経理担当者に電話。担当者は本物と思い込み、約3,300万円を送金。

◆ AI生成フィッシングメール（AIを使った文章の偽造）

手法の本質：AIを使って自然な文章の詐欺メールを大量生成する攻撃。従来の「怪しい日本語」が消え、見抜くのが難しくなる。

従来のフィッシングメールAI生成フィッシングメール
文法ミス・不自然な日本語自然な文法・敬語
一律の内容ターゲットごとに最適化
見抜きやすい本物と区別が困難
スパム判定で検知されやすい従来のスパム判定が通用しにくい

AI生成フィッシングの実例

  • ChatGPT悪用フィッシング（2023年・世界各国）：攻撃者がChatGPTを使って自然な日本語のフィッシングメールを大量生成していた。
  • BEC（ビジネスメール詐欺）でAI文体模倣（2023〜2024年）：攻撃者がAIに「上司の文体」を学習させ、本物そっくりのメールを生成。複数企業で実際に送金被害が発生。

◆ 攻撃の自動化・最適化（AIによる攻撃作業の高速化）

手法の本質：AIを使って攻撃の準備・実行・調整を自動化する攻撃。人間が手作業でやっていたことをAIが24時間・大規模に実行する。

実例詳細：BISS SCANNERによる900社侵害事件（2025〜2026年）

2026年4月、DFIR REPORTの研究者が攻撃者のサーバが誤って公開状態になっているのを発見。そこには以下が残っていた：

  • 900社以上の企業から盗まれた30,000件以上の認証情報
  • 攻撃の自動化に使われたAIツール（CLAUDE CODE / OPENCLAW）
  • 7ヶ月間稼働していた攻撃ログ
  • 自動スキャン→資格情報収集→仕分け→送信の完全自動パイプライン

何がAIだったのか：AIがコード解析・修正を自動化。AIが攻撃パイプライン全体をオーケストレーション。1人の攻撃者が「本来はチーム規模の攻撃」を実行できた。

被害：900社以上が侵害。APIキー、クラウド認証情報、AIプラットフォームの鍵などが大量流出。

実例詳細：大規模自動スキャン攻撃（2024〜2025年）

AIと自動化を組み合わせた「大規模自動スキャン攻撃」が世界的に急増。

  • 毎秒36,000回のスキャン
  • 数十億件のスキャンが毎月観測
  • 攻撃者がAIを使って「脆弱なサービス」を高速に特定

結果：公開アプリケーションの脆弱性悪用が44%増加。中小企業を中心に多数の侵害が発生。

実例詳細：AIによる資格情報大量窃取（2025年）

IBMの2026年レポートで、AIを使った資格情報窃取攻撃が急増していると報告。

  • 30万件以上のChatGPTアカウント情報が流出
  • 攻撃者がAIを使って脆弱な公開アプリを高速に特定
  • そこから資格情報を盗む攻撃が増加

まとめ：AIが攻撃のスピードと規模を劇的に増加させている。

間違えやすいポイント

  • 「ディープフェイク＝画像だけ」ではない：音声も生成可能。CEOの声を模倣した送金詐欺の実例がある。
  • 「AIフィッシングは日本語が不自然」という前提は崩れた：現代のAIは敬語やビジネス文体を自然に生成できる。
  • 「攻撃自動化は未来の話」ではない：BISS SCANNER事件のように2025〜2026年時点で実際に発生している。
  • 「1人の攻撃者が小規模」という前提は崩れた：AIを使うことで1人の攻撃者が従来はチーム規模だった攻撃を実行可能。
  • プロンプトインジェクションとの違い：プロンプトインジェクションはAIへ不正な指示を与える攻撃。AI生成フィッシングはAIを使ってメールを生成する攻撃。

ア）大量のメール・画像・音声を生成できる　イ）人間では気づけない精度で偽装できる　ウ）攻撃の自動化・高速化が可能　エ）AIは必ず無料で利用できる

答えを見る ▼

答え: エ — AIの利用コストは攻撃の動機としては重要だが、「必ず無料」というのは誤り。他の選択肢はすべてPDFに記載されている理由である。

Q2. ディープフェイクの生成プロセスとして正しい順序はどれか。

ア）AI学習 → 素材収集 → 上書き合成 → 調整　イ）素材収集 → AI学習 → 上書き合成 → 調整　ウ）上書き合成 → 素材収集 → AI学習 → 調整　エ）調整 → AI学習 → 素材収集 → 上書き合成

答えを見る ▼

答え: イ — 正しい順序は「素材収集（顔・声データ）→ AI学習（特徴を学習）→ 上書き合成（別人の映像に合成）→ 調整（違和感を除去）」である。

Q4. AI生成フィッシングメールの特徴として適切でないものはどれか。

Q6. AIによる攻撃自動化の例として適切でないものはどれか。
