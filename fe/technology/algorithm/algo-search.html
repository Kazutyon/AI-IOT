2. 線形探索（Linear Search）

線形探索（リニアサーチ）は、配列の先頭から順に、1つずつ値を調べていく探索方法です。探索アルゴリズムの中で最も基本的であり、整列されていないデータにもそのまま使えます。

擬似コード

function linear_search(a, n, x):
    i = 0
    while i < n:
        if a[i] == x:
            return i       // 見つかった！
        i = i + 1
    return -1              // 見つからなかった

計算量: O(n) — 要素数に比例して処理時間が増える。整列不要、実装が単純。見つかった時点で探索終了。

3. 二分探索（Binary Search）

二分探索（バイナリサーチ）は、整列された配列を対象に、探索範囲を半分ずつ狭めていく探索です。「真ん中を見る→範囲を半分に絞る」を繰り返します。

前提条件（最重要）

配列が昇順または降順に整列されていること。未整列配列に二分探索を適用すると正しい結果が得られません。FE試験頻出。

擬似コード

function binary_search(a, n, x):
    left = 0
    right = n - 1
    while left <= right:
        mid = (left + right) // 2
        if a[mid] == x:
            return mid
        if a[mid] < x:
            left = mid + 1
        else:
            right = mid - 1
    return -1

計算量: O(log n) — 要素数100万でも約20回の比較で済む。毎回範囲が約1/2になるため: n→n/2→n/4→...→1。

4. 線形探索 vs 二分探索

項目線形探索二分探索
配列の整列不要（どんな並びでも可）必須（昇順/降順）
調べ方先頭から順に1つずつ中央→範囲を半分に絞る
計算量O(n)O(log n)
要素増加時比例して遅くなる増えても増加が小さい
コード簡単（数行）やや複雑（範囲管理）
使いどころ小規模・未整列大規模・整列済み
100万件の比較最大1,000,000回約20回

どちらを使うべきか

  • 未整列→線形探索 / 大規模整列済み→二分探索
  • 小規模→線形でも十分 / 探索1回→線形、何度も→整列して二分

確認問題

Q1. 線形探索の計算量はどれか。

ア）O(1)　イ）O(log n)　ウ）O(n)　エ）O(n²)

答えを見る ▼

答え: ウ — 配列の先頭から順に1つずつ調べるため、最悪ケースで全要素走査し O(n)。

Q2. 二分探索の必須前提条件はどれか。

ア）データが連結リスト　イ）データが整列されている　ウ）ハッシュ表に格納　エ）スタック構造

答えを見る ▼

答え: イ — 二分探索は整列が絶対条件。未整列データには使えない。

Q3. 要素数1,000,000の整列済み配列で二分探索の最大比較回数は約何回か。

ア）約10回　イ）約20回　ウ）約1,000回　エ）約1,000,000回

答えを見る ▼

答え: イ — log₂(1,000,000) ≒ 20。毎回範囲が半分になるため。

Q4. 配列 [2,4,5,7,9,12,15] で値12を二分探索するとき最初の中央値はどれか。

ア）2　イ）5　ウ）7　エ）12

答えを見る ▼

答え: ウ — mid=(0+6)//2=3, a[3]=7。7<12 なので右半分へ。

Q5. 線形探索で値が見つからなかった場合の一般的な戻り値はどれか。

ア）0　イ）null　ウ）-1　エ）末尾のインデックス

答えを見る ▼

答え: ウ — 配列インデックスは0以上なので -1 で「見つからなかった」を表す。

Q6. 二分探索の計算量が O(log n) になる理由はどれか。

ア）全要素を1回ずつ調べるから　イ）毎回探索範囲が約半分になるから　ウ）ハッシュ関数で計算するから　エ）並列処理するから

答えを見る ▼

答え: イ — mid比較後、左半分/右半分に絞るため範囲が毎回約1/2になる。

Q7. 未整列の配列 [8,3,1,6,4] から値6を探す場合、適切な方法はどれか。

ア）二分探索　イ）ハッシュ探索　ウ）線形探索　エ）幅優先探索

答えを見る ▼

答え: ウ — 未整列のため二分探索不可。線形探索は整列不要でそのまま使える。

Q8. 二分探索で left > right となった場合の意味はどれか。

ア）探索続行　イ）値が見つかった　ウ）値が存在しない　エ）先頭に戻る

答えを見る ▼

答え: ウ — 探索範囲消滅 = 値が存在しない。return -1 を返す。
