1. フローチャート（流れ図）

フローチャート（流れ図）は、アルゴリズムを図式で表現したものです。処理の流れを視覚化することで、プログラムを書く前にアルゴリズムの正しさを確認できます。記号はJIS規格で定められており、FE試験でも頻出です。

2. 再帰関数

再帰（Recursion）とは、関数が自分自身を呼び出すプログラミング技法です。大きな問題を「同じ形の小さな問題」に分割して解くことで、複雑な処理を簡潔に表現できます。

2-1. 再帰の2大要素

再帰関数には必ず次の2つが必要です。

  • 基底条件（停止条件）：再帰を停止する条件。これがないと無限に自分を呼び続け、スタックオーバーフローになる。典型的には n=0 や n=1。
  • 再帰条件：自分自身を呼び出す条件。問題を1つ小さいサイズ（例: n→n-1）に縮小し、基底条件に向かって収束させる。

2-2. コールスタックと再帰の動作

再帰呼び出しのたびに、関数の情報（引数・戻り先・ローカル変数）がコールスタックに積まれます。基底条件に達すると、スタックを巻き戻しながら計算結果が伝播していきます。

2-3. 再帰 vs 反復（ループ）の比較

項目再帰反復（ループ）
実現方法関数が自分自身を呼ぶfor / while で繰り返す
コードの簡潔さ簡潔（問題によっては劇的に）やや長くなる
メモリ使用コールスタックを使う（深さに比例）スタックを使わない（メモリ効率良）
スタックオーバーフロー深すぎると危険発生しない
適する問題木構造・分割統治・DFS単純な繰り返し

3. 再帰の例：階乗とフィボナッチ

3-1. 階乗（n!）の再帰

階乗 n! は「1からnまでの整数をすべて掛けた値」です。再帰定義は次のとおり。

  • 基底条件: n = 0 のとき 0! = 1
  • 再帰条件: n ≧ 1 のとき n! = n × (n-1)!

3-2. フィボナッチ数列の再帰

フィボナッチ数列は「前の2つの項の和で次の項を作る」数列です。再帰定義は次のとおり。

  • 基底条件: F(0) = 0, F(1) = 1
  • 再帰条件: F(n) = F(n-1) + F(n-2) （n ≧ 2）
項目階乗 fact(n)フィボナッチ fib(n)（素朴再帰）
再帰呼び出し回数1回（n→n-1の一本道）2回（n→n-1 と n→n-2 の二股）
計算量O(n)O(2ⁿ)
重複計算なしあり（同じ値を何度も計算）
n=50での目安約50回約1.1×10¹⁵（非現実的）

4. 計算量とBig-O記法

計算量とは、アルゴリズムの性能を表す尺度です。入力サイズnが大きくなったとき、実行時間やメモリ使用量がどのように増えるかを評価します。

4-1. Big-O記法のルール

Big-O記法（O記法）は計算量のオーダー（概略的な増加率）を表します。ルールは次の2つ。

  1. 定数倍を無視する：3n²も100n²も同じ O(n²)。nが大きくなれば定数の差は無視できる。
  2. 低次の項を無視する：n² + 100n + 1000 は O(n²)。nが大きくなれば n² の項が支配的になる。

例えば、あるアルゴリズムのステップ数が 3n² + 5n + 200 の場合、Big-Oでは O(n²) と表現します。

4-2. Big-O 早見表

O記法名称増え方（nが2倍で）代表的なアルゴリズム
O(1)定数時間変わらない配列のindexアクセス、ハッシュ表検索
O(log n)対数時間+1 増加二分探索、BSTの探索
O(n)線形時間2倍線形探索、配列の合計計算
O(n log n)準線形時間約2倍+αマージソート、クイックソート（平均）

O(2ⁿ)指数時間2乗フィボナッチ素朴再帰、組合せ全探索

確認問題

Q1. フローチャートにおいて、条件分岐を表す記号はどれか。

ア）長方形　イ）ひし形　ウ）角丸長方形　エ）六角形

答えを見る ▼

答え: イ — 判断記号はひし形で、条件分岐（Yes/No）を表す。長方形は処理記号、角丸長方形は端子記号、六角形はループ端記号。

Q2. フローチャートでサブルーチン（関数呼び出し）を表す記号はどれか。

ア）ひし形　イ）長方形　ウ）二重線の長方形　エ）矢印

答えを見る ▼

答え: ウ — 定義済み処理記号（二重線の長方形）は、別の場所で定義されたサブルーチン（関数）の呼び出しを表す。

Q3. 再帰関数において、無限ループを防ぐために必ず必要なものはどれか。

ア）ループ端記号　イ）グローバル変数　ウ）基底条件（停止条件）　エ）戻り値

答えを見る ▼

答え: ウ — 基底条件（停止条件）がないと再帰呼び出しが無限に続き、スタックオーバーフローになる。n=0やn=1が典型。

Q4. 次の再帰関数 fact(4) の戻り値はいくつか。fact(n) = 1 (n=0), n × fact(n-1) (n≧1)

ア）16　イ）20　ウ）24　エ）120

答えを見る ▼

答え: ウ — 4! = 4 × 3 × 2 × 1 = 24。fact(4) → 4×fact(3) → 4×3×fact(2) → 4×3×2×fact(1) → 4×3×2×1×fact(0)=4×3×2×1×1=24。

Q5. fact(0) の値はいくつと定義されているか。

ア）0　イ）1　ウ）未定義　エ）-1

答えを見る ▼

答え: イ — 0! = 1 と定義されている。これが階乗の再帰における基底条件になる（n=0で停止して1を返す）。

Q6. フィボナッチ数列 F(5) の値はいくつか。F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)

ア）3　イ）5　ウ）8　エ）13

答えを見る ▼

答え: イ — F(0)=0, F(1)=1, F(2)=1+0=1, F(3)=1+1=2, F(4)=2+1=3, F(5)=3+2=5。

Q7. 素朴な再帰で実装した fibonacci(n) の時間計算量を示すO記法はどれか。

ア）O(n)　イ）O(log n)　ウ）O(n²)　エ）O(2ⁿ)

答えを見る ▼

答え: エ — フィボナッチの素朴再帰では1回の呼び出しが2回の再帰呼び出しを生むため、計算量はO(2ⁿ)。同じ値の重複計算が大量に発生する。

Q8. 再帰と反復（ループ）の比較として正しいものはどれか。

ア）再帰は常に反復より高速である　イ）再帰はコールスタックを使うため深すぎるとスタックオーバーフローの危険がある　ウ）反復は再帰よりコードが常に短い　エ）再帰はメモリを一切使わない

答えを見る ▼

答え: イ — 再帰は呼び出しのたびにコールスタックを消費するため、深さが深すぎるとスタックオーバーフローになる。反復にはこの危険がない。

Q9. 線形探索の平均時間計算量を示すO記法はどれか。

ア）O(1)　イ）O(log n)　ウ）O(n)　エ）O(n²)

答えを見る ▼

答え: ウ — 線形探索は先頭から順に探すため、平均でn/2回、最悪でn回の比較が必要。計算量はO(n)。

Q10. 二分探索の時間計算量を示すO記法はどれか。

ア）O(1)　イ）O(log n)　ウ）O(n)　エ）O(n log n)

答えを見る ▼

答え: イ — 二分探索は毎回探索範囲が約1/2になるため、2^k = n → k = log₂ n より O(log n)。整列済み配列が前提。

Q11. バブルソートの時間計算量を示すO記法はどれか。

ア）O(n)　イ）O(log n)　ウ）O(n log n)　エ）O(n²)

答えを見る ▼

答え: エ — バブルソートは二重ループ（外側n回×内側n回）で全要素を比較するため O(n²)。選択ソート・挿入ソートも同じ。

Q12. 次のうち、最も効率の良い計算量はどれか。

ア）O(1)　イ）O(log n)　ウ）O(n)　エ）O(n²)

答えを見る ▼

答え: ア — O(1) は定数時間で、nがいくら大きくなっても実行時間は変わらない。効率順: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)。

Q13. Big-O記法で「3n² + 5n + 100」はどのように表されるか。

ア）O(3n²)　イ）O(n² + n)　ウ）O(n²)　エ）O(1)

答えを見る ▼

答え: ウ — Big-Oでは定数倍（3）と低次の項（5n, 100）を無視する。nが十分大きいとき n² の項が支配的になるため O(n²) と表す。

Q14. マージソートの時間計算量を示すO記法はどれか。

ア）O(n)　イ）O(log n)　ウ）O(n log n)　エ）O(2ⁿ)

答えを見る ▼

答え: ウ — マージソートは分割（log n段階）× 統合（各段階n回）で O(n log n)。クイックソートの平均も同じ。O(n²)のソートより高速。

Q15. フローチャートで処理の開始と終了を示す記号の形はどれか。

ア）長方形　イ）ひし形　ウ）角丸長方形　エ）六角形

答えを見る ▼

答え: ウ — 端子記号（角丸長方形）はアルゴリズムの開始（Start）と終了（End）を表す。長方形は処理、ひし形は判断、六角形はループ端。
