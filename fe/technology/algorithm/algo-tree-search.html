1. 木構造とは何か

木構造（Tree）は、1つの根（root）から始まり、親→子という階層構造を持つデータ構造です。配列や連結リストでは階層を表現できないため、フォルダ構造・HTML DOM・組織図・決定木・データベースのインデックス（B木）などに使われます。

木はグラフの一種ですが、次の3つの制約があります：①閉路（ループ）がない ②親は1つだけ ③根が1つだけ。この制約があるからこそ、木は階層構造を表現するのに最適です。

2. 二分木（Binary Tree）

二分木は、1つのノードが「左の子」「右の子」の最大2つまで子を持つ木構造です。木構造の中でも最も基本的で、後のBST・ヒープ・探索アルゴリズムの基礎になります。

各ノードは key（値）、left（左の子への参照）、right（右の子への参照）の3要素を持ちます。

2-1. 満二分木 vs 完全二分木

項目満二分木完全二分木
定義全てのノードが0個または2個の子を持つ最下段以外は全て埋まり、最下段は左から詰まっている
子の数0個または2個のみ（1個の子はNG）最下段は左から順に埋まる（右が欠けてもOK）
最下段制約なし（葉はどの段でもOK）最下段の葉は左詰め
使用場面理論的な分類ヒープの実装

2-2. 深さと高さ

深さ（depth）は根からの距離（根を0とする）。高さ（height）は葉からの距離（葉を0とする）。この2つはFE試験で混同しやすいので注意。

深さ0: [A] ←根（depth=0, height=2）
         /   \
深さ1: [B]   [C] ←葉（depth=1, height=0）
      /   \
深さ2:[D] [E] ←葉（depth=2, height=0）

3. 二分探索木（BST）

二分探索木（Binary Search Tree）は、左部分木の値 < 親ノードの値 < 右部分木の値という順序ルールを持つ二分木です。このルールが全ての部分木で成立することが重要です。

3-1. BSTの挿入（insert）

BSTの挿入は「根から比較→左か右へ→空き場所に追加」の繰り返しです。毎回1回の比較で探索範囲が約半分になるため、平均 O(log n) で挿入できます。

3-2. BSTの探索（search）

search(7) の流れ：根(8)と比較→7<8で左→3と比較→7>3で右→6と比較→7>6で右→7と一致！比較回数は木の高さに比例します。

3-3. BSTが高速な理由 O(log n)

挿入も探索も毎回「左か右か」の1回の比較で進み、探索範囲が毎回1/2になります。n→n/2→n/4→n/8→...→1、つまり 2^k = n → k = log₂(n) の形です。ただし偏った木（片側にだけ伸びる）では最悪 O(n) になることに注意。

4. DFS（深さ優先探索）

DFSは「深く潜ってから戻る」探索方法です。木構造では根→左の部分木→右の部分木の順で進みます。DFSには3種類の訪問順があります。

4-1. DFS（前順）のコード

function preorder(node):
    if node is null: return
    visit(node)              // 親を処理
    preorder(node.left)      // 左の部分木へ
    preorder(node.right)     // 右の部分木へ

DFSは再帰で簡潔に書けます。反復版ではスタックを使います。

走査種別訪問順BSTでの性質主な用途
前順親→左→右木の構造そのまま木のコピー
中順左→親→右昇順ソート順BSTのソート出力
後順左→右→親子から先に処理木の削除

5. BFS（幅優先探索）

BFSは「横に広がりながら進む」探索方法です。DFSが「深く潜る」のに対し、BFSはレベル（階層）ごとに探索します。

5-1. BFSがキューを使う理由

BFSは「先に見つけたノードから順に処理する」性質を持ちます。これはキュー（FIFO: First-In First-Out）の動作と一致するため、BFSは必ずキューを使います。

5-2. BFSのコード

function bfs(root):
    if root is null: return
    queue = new Queue()
    queue.enqueue(root)
    while queue is not empty:
        node = queue.dequeue()
        visit(node)
        if node.left is not null:
            queue.enqueue(node.left)
        if node.right is not null:
            queue.enqueue(node.right)

6. DFS vs BFS 比較

項目DFS（深さ優先）BFS（幅優先）
進み方深く潜ってから戻る横に広がってから下へ
使用データ構造再帰 または スタック（LIFO）キュー（FIFO）
訪問順序部分木の構造を優先距離（レベル）を優先
主な用途木構造の走査・BSTソート出力最短経路探索・レベル順出力
メモリ使用木の高さに比例（再帰深度）木の幅に比例（キュー容量）

7. DFS中順 = BST昇順（最重要ポイント）

FE試験で最も差がつく論点です：

  • BSTのルール: 左 < 親 < 右
  • 中順DFSの順番: 左 → 親 → 右
  • この2つが完全に一致する → 中順DFS = 昇順ソートになる

つまり「BSTを中順走査すると、値が小さい順に取り出せる」。これは木構造（形）と探索（順番）が結びつく最も重要なポイントで、午前試験の選択問題でも頻出です。

8. 木の探索からグラフ探索へ

木は「閉路のないグラフ」として扱えます。DFSもBFSも、木構造だけでなくグラフ探索の基本アルゴリズムです。グラフでは閉路があるため visited 管理が追加で必要になります（第4章で詳しく扱います）。

確認問題

Q1. 二分木において、子を持たないノードを何と呼ぶか。

ア）根　イ）内部ノード　ウ）葉　エ）親

答えを見る ▼

答え: ウ — 葉（leaf）は子を持たないノード。根は親を持たないノード、内部ノードは子を持つノード。

Q2. 二分探索木（BST）の正しい定義はどれか。

ア）左の子 < 親 かつ 親 < 右の子（全部分木で成立）
イ）左の子 > 親 かつ 親 > 右の子
ウ）全てのノードが2つの子を持つ
エ）最下段が左から詰まっている

答えを見る ▼

答え: ア — BSTは「左部分木 < 親 < 右部分木」が全部分木で成立する二分木。イは逆、ウは満二分木、エは完全二分木の説明。

Q3. 幅優先探索（BFS）で使用するデータ構造はどれか。

ア）スタック　イ）キュー　ウ）配列　エ）ハッシュ表

答えを見る ▼

答え: イ — BFSは「先に見つけたノードから順に処理」するためFIFOのキューを使う。DFSは再帰またはスタック（LIFO）を使う。

Q4. 次の木を前順走査（親→左→右）した結果を選べ。A─(B─(D,E), C) の場合。

ア）A, B, C, D, E　イ）A, B, D, E, C　ウ）D, B, E, A, C　エ）D, E, B, C, A

答えを見る ▼

答え: イ — 前順（親→左→右）: A（親）→ B（左）→ D（Bの左）→ E（Bの右）→ C（Aの右）。アは不正、ウは中順、エは後順。

Q5. 二分探索木において中順走査を行うと得られる結果はどれか。

ア）後順の逆順　イ）ランダムな順序　ウ）昇順ソートされた順序　エ）レベル順

答えを見る ▼

答え: ウ — BSTの「左<親<右」と中順DFSの「左→親→右」が完全一致するため、中順走査 = 昇順ソートになる。これはFE試験最重要論点。

Q6. 満二分木の説明として正しいものはどれか。

ア）全ての葉が同じ深さにある　イ）全てのノードが0個または2個の子を持つ　ウ）最下段が左から詰まっている　エ）全てのノードが2つの子を持つ

答えを見る ▼

答え: イ — 満二分木は全ノードが0個または2個の子を持つ。アは完全二分木の条件ではない、ウは完全二分木、エは子を持たない葉が存在するため誤り。

Q7. BSTに [8,3,10,1,6,14,4,7,16] を順に挿入したとき、ノード6の左の子はどれか。

ア）1　イ）4　ウ）7　エ）なし

答えを見る ▼

答え: イ — insert(4)のとき 4<8→左、4>3→右、4<6→左 でノード6の左の子になる。7は6の右の子、1は3の左の子。

Q8. 深さ優先探索（DFS）で使うデータ構造として最も適切なものはどれか。

ア）キュー　イ）スタック（または再帰）　ウ）ハッシュ表　エ）連結リスト

答えを見る ▼

答え: イ — DFSは再帰呼び出し（システムスタック）または明示的なスタックで実装する。BFSはキューを使う。

Q9. BSTの探索が平均O(log n)になる理由として最も適切なものはどれか。

ア）全てのノードを一度ずつ訪問するから　イ）毎回探索範囲が約1/2になるから　ウ）キューを使って効率的に探索するから　エ）ハッシュ関数で直接計算するから

答えを見る ▼

答え: イ — BSTは毎回「左か右か」の比較で探索範囲が約1/2になる。n→n/2→n/4→...→1 で 2^k=n → k=log₂n。ただし偏った木では最悪O(n)。

Q10. 次のうち、木構造とグラフ構造の違いとして正しいものはどれか。

ア）木には閉路がないが、グラフには閉路があってもよい　イ）グラフにはノードがない　ウ）木は必ず完全二分木である　エ）グラフは根を必ず1つ持つ

答えを見る ▼

答え: ア — 木は「閉路のないグラフ」であり、閉路なし・親1つ・根1つの制約がある。グラフにはこれらの制約がない。イ・ウ・エはいずれも誤り。
