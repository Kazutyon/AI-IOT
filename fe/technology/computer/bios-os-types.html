リアルタイムOS（RTOS） 決められた時間内に必ず処理を完了することを重視したOS FreeRTOS、VxWorks、μITRON。工場ロボット、医療機器、ドローンなど
IoT向けOS IoTデバイス向けに最適化された軽量OS Zephyr、Mbed OS、RIOT OS。超省電力・小メモリで動作
車載OS 自動車のECUや自動運転システムで利用されるOS AUTOSAR、QNX、AGL。高信頼性・安全規格（ISO 26262）対応
Windows マイクロソフト社の汎用OS。多様なハードウェアで動作 後方互換性を重視。企業向け管理機能が充実
macOS Apple社の汎用OS。UNIXベース Apple製ハード専用に最適化。安定性・UI/UXを重視

3.1 コンピュータの起動と BIOS / UEFI

コンピュータは電源を入れるとすぐに OS が動き出すわけではありません。最初に動作するのは、マザーボードに組み込まれた BIOS または UEFI と呼ばれるプログラムです。

BIOS / UEFI の役割

役割 説明
ハードウェアの初期化 CPU・メモリ・ストレージ・キーボードなどを認識し、使える状態にする
POST（Power-On Self Test） 電源投入時に、ハードウェアが正常に動作しているかをチェックする
ブートローダの起動 OSを読み込むためのプログラム（ブートローダ）をストレージから探し、実行する

BIOSとUEFIの違い

組み込み・IoT におけるブート

項目 BIOS UEFI
方式 古い方式（1980年代から） 新しい方式（2000年代から）
画面 テキストベース グラフィカル
起動方式 MBR（Master Boot Record） GPT（GUID Partition Table）
メモリ利用 1MB以下 大容量利用可能
セキュリティ 弱い Secure Bootなど強化
項目 内容
代表的なブートローダ U-Boot（LinuxベースのIoT機器で標準）
特徴 小型・省メモリ・高速起動・安定性重視
ファームウェア更新 OTA（Over The Air）による無線更新が一般的
セキュリティ 署名付きファームウェアによる改ざん防止

3.2 OS（オペレーティングシステム）とは

OSは、ハードウェアを管理し、アプリケーションが動作できる環境を提供するソフトウェアです。

OSの3つの役割

役割 説明 具体例
資源管理 コンピュータの各資源を管理し、アプリケーションに適切に割り当てる CPU（プロセス管理）、メモリ（メモリ管理）、ストレージ（ファイル管理）、入出力装置（デバイス管理）
抽象化 ハードウェアの複雑さを隠し、アプリケーションが統一的な方法で操作できるようにする 「ファイル」という概念でストレージを扱う。「プロセス」という概念でCPUを扱う
ユーザインターフェース ユーザがコンピュータを操作できる環境を提供する GUI（Windows、macOS、Android）、CUI（Linuxのシェルなど）

OSの種類

① 汎用OS（General Purpose OS）

OS 特徴 用途
Windows 多様なハードウェアで動作。後方互換性重視。企業向け管理機能充実 PC、サーバ、業務用端末
macOS Apple製ハード専用に最適化。UNIXベース。安定性・UI/UX重視 Mac（デスクトップ・ノートPC）
Linux オープンソース。多様なディストリビューション（Ubuntu、CentOSなど） サーバ、開発環境、組み込み、スパコン

② モバイルOS

OS 特徴 用途
iOS Apple製iPhone/iPad専用。タッチ操作に最適化。App Storeで管理 iPhone、iPad
Android Google開発。多様なメーカーのデバイスで動作。オープンソースベース スマートフォン、タブレット

③ リアルタイムOS（RTOS）

OS 特徴 用途
FreeRTOS オープンソース。軽量・無料。組み込みで最も広く利用 家電、IoTデバイス、ドローン
VxWorks 商用RTOS。高い信頼性とリアルタイム性 航空宇宙、産業機器、医療機器
μITRON 日本で広く使われたRTOS規格。自動車や家電で実績 車載機器、デジカメ、プリンタ

④ IoT向けOS

OS 特徴 用途
Zephyr Linux財団開発。超省電力・小メモリ（数十KBから） BLE、LoRa、センサーネットワーク
Mbed OS Arm開発。クラウド連携が容易 スマートホーム、ウェアラブル
RIOT OS 低消費電力。IoTデバイス向けに特化 環境センサー、スマートシティ

⑤ 車載OS（Automotive OS）

十九章インフォテインメントシステム
OS 特徴 用途
AUTOSAR 自動車業界標準のOS規格。Classic（制御系）とAdaptive（情報系） エンジン制御、ブレーキ制御、ADAS
QNX 商用RTOS。高い安全性（ISO 26262認証） 自動運転システム、デジタルコックピット
AGL（Automotive Grade Linux） オープンソースの車載OS。Linuxベース

⑥ マイコン向けOS

OS 特徴 用途
Arduino Core Arduinoマイコン向けの軽量フレームワーク 電子工作、プロトタイピング
FreeRTOS（MCU版） マイコンでも動作する軽量RTOS 家電制御、小型ロボット

WindowsとmacOSの設計思想の違い

OS 設計思想 特徴
Windows 汎用性・後方互換性・企業利用を最優先 多様なハードウェアで動作。自作PCから企業端末まで。古いアプリも動かすため複雑化
macOS 統一されたハードウェアでの最適化・安定性・UXを最優先 Apple製ハード専用。UNIXベースで設計がシンプル。UI/UXの一貫性を重視

確認問題

Q1. BIOSの役割として、適切でないものはどれか。

ア）ハードウェアの初期化　イ）POST（自己診断テスト）の実行　ウ）アプリケーションの実行　エ）ブートローダの起動

答えを見る ▼

答え: ウ — BIOSはOS起動前の処理を行い、アプリケーションの実行はOSの役割。

Q2. UEFIの特徴として、適切なものはどれか。

ア）テキストベースのインターフェース　イ）MBR方式の起動　ウ）Secure Bootなどのセキュリティ機能　エ）1MB以下のメモリ制限

答えを見る ▼

答え: ウ — UEFIはSecure Bootなどのセキュリティ機能を持つ。ア、イ、エはBIOSの特徴。

Q3. OSの役割として、本文で挙げられていないものはどれか。

ア）資源管理　イ）抽象化　ウ）ユーザインターフェース　エ）アプリケーションの開発

答えを見る ▼

答え: エ — OSの3つの役割は「資源管理」「抽象化」「ユーザインターフェース」。アプリケーションの開発は開発者の役割。

Q4. リアルタイムOS（RTOS）の特徴として、最も適切なものはどれか。

ア）多機能で大規模なアプリケーション向け　イ）決められた時間内に必ず処理を完了するリアルタイム性を重視　ウ）タッチ操作に最適化　エ）オフィス業務向け

答えを見る ▼

答え: イ — RTOSは決められた時間内に必ず処理を完了するリアルタイム性を重視。工場ロボットや医療機器で利用。

Q5. 以下のうち、IoT向けOSはどれか。

ア）Windows 11　イ）macOS　ウ）Zephyr　エ）iOS

答えを見る ▼

答え: ウ — ZephyrはIoT向けOS。他は汎用OSまたはモバイルOS。

Q6. BIOSと比較したUEFIの特徴として、適切なものはどれか。

ア）テキストベースの画面　イ）MBR方式の起動　ウ）グラフィカルなインターフェースと大容量メモリ利用可能　エ）1MB以下のメモリ制限

答えを見る ▼

答え: ウ — UEFIはグラフィカルなインターフェースを持ち、大容量メモリを利用可能。ア、イ、エはBIOSの特徴。

Q7. OSにおける「抽象化」の例として、適切なものはどれか。

ア）CPUの使用率を監視する　イ）「ファイル」という概念でストレージを扱う　ウ）メモリ容量を増設する　エ）ディスクをフォーマットする

答えを見る ▼

答え: イ — 抽象化の例は「ファイル」という概念でストレージを扱うこと。ハードウェアの複雑さを隠し、統一的な操作を提供する。

Q8. 車載OSの代表例として、適切なものはどれか。

ア）Windows 11　イ）Android　ウ）AUTOSAR　エ）FreeRTOS

答えを見る ▼

答え: ウ — AUTOSARは自動車業界標準の車載OS。FreeRTOSは汎用RTOSだが車載でも使われる場合がある。

Q9. POST（Power-On Self Test）の説明として、適切なものはどれか。

ア）OSの起動後に実行されるテスト　イ）電源投入時にハードウェアが正常に動作するかチェックするテスト　ウ）アプリケーションの動作テスト　エ）ネットワーク接続テスト

答えを見る ▼

答え: イ — POSTは電源投入時にBIOS/UEFIが実行するハードウェアの自己診断テスト。

Q10. ブートローダの役割はどれか。

ア）アプリケーションを終了する　イ）OSを読み込むためのプログラムを起動する　ウ）ファイルを圧縮する　エ）ネットワーク設定を行う

答えを見る ▼

答え: イ — ブートローダはOSを読み込むためのプログラム（Windows Boot Manager、GRUB、U-Bootなど）をストレージから探し、実行する。
