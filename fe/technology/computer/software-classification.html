7.1 システムソフトウェア（詳細）

システムソフトウェアは、コンピュータを動作させるための基盤となるソフトウェアであり、アプリケーションが動作するための環境を提供します。

種類説明代表例
OS（オペレーティングシステム）ハードウェア資源を管理し、アプリケーションが動作する基盤を提供Windows、Linux、macOS、Android、iOS
デバイスドライバOSとハードウェアをつなぐソフトウェアUSBドライバ、ネットワークドライバ、グラフィックドライバ
ファームウェアハードウェアを直接制御するために機器内部に組み込まれたソフトウェアBIOS/UEFI、マイコン制御プログラム、SSD内部制御
RTOS（リアルタイムOS）リアルタイム性を重視したOS。タスクの優先度が厳密に守られるFreeRTOS、VxWorks、Zephyr、μITRON

7.2 ミドルウェア

ミドルウェアは、OSとアプリケーションの間で共通機能を提供するソフトウェアであり、アプリケーション開発を大幅に効率化します。

ミドルウェアの種類

種類説明代表例
DBMSデータの永続化、同時アクセス制御、検索高速化、障害復旧MySQL、PostgreSQL、SQLite
WebサーバHTTPリクエスト受付、静的ファイル配信、リバースプロキシApache、Nginx、Lighttpd
アプリケーションサーバアプリケーションの実行環境を提供Node.js、Tomcat、Django/Flask
APIゲートウェイ認証・認可、ルーティング、レート制限、ログ収集Kong、Amazon API Gateway、Nginx
メッセージング（MQTT）非同期でデータを送受信する仕組み。IoTで標準的に利用MQTT、RabbitMQ、Kafka
AIミドルウェアニューラルネットワークの構築、学習・推論の高速化TensorFlow、PyTorch、ONNX Runtime
ロボットミドルウェア（ROS2）センサー・アクチュエータ統合、ノード間通信、経路計画ROS2（Robot Operating System 2）
IoTミドルウェアデバイス認証、MQTT通信、データ収集、クラウド連携AWS IoT Core、Azure IoT Hub、Google Cloud IoT
セキュリティミドルウェア暗号化・認証・アクセス制御を提供OpenSSL、Keycloak、JWT

7.3 アプリケーションソフトウェア

アプリケーションソフトウェアは、ユーザの目的に応じて動作するソフトウェアであり、OSやミドルウェアの上で動作します。

種類説明代表例
業務アプリケーション企業の業務を支援。データベース連携が前提会計ソフト、販売管理システム、CRM、ERP
WebアプリケーションWebブラウザ上で動作。HTTP/HTTPSでサーバと通信Google Workspace、SNS、ECサイト、Webメール
モバイルアプリスマートフォンやタブレットで動作。GPS・カメラなどと連携iOS/Androidアプリ、地図アプリ、SNSアプリ
AIアプリケーション機械学習モデルを利用。画像認識、音声認識、翻訳など画像認識アプリ、音声アシスタント、翻訳アプリ
IoTアプリケーションIoTデバイスと連携。MQTT・REST APIで状態・コマンドをやり取りスマートスピーカー、遠隔診療、カーシェアアプリ
デスクトップアプリケーションPC上で動作。ローカルリソースを直接利用Microsoft Office、Photoshop、Visual Studio Code
組み込みアプリケーション家電・車載機器などに組み込まれた専用アプリ。リアルタイム性重視配膳ロボット、車載ナビ、エレベーター制御

確認問題

Q1. ソフトウェアの3層分類に含まれないものはどれか。

ア）システムソフトウェア　イ）ミドルウェア　ウ）アプリケーションソフトウェア　エ）ハードウェア

答えを見る ▼

答え: エ — ソフトウェアの3層分類は「システムソフトウェア」「ミドルウェア」「アプリケーションソフトウェア」。ハードウェアは別物。

Q2. システムソフトウェアに分類されるものはどれか。

ア）Webブラウザ　イ）Excel　ウ）OS、デバイスドライバ、ファームウェア　エ）スマホアプリ

答えを見る ▼

答え: ウ — システムソフトウェアにはOS、デバイスドライバ、ファームウェア、RTOSなどが含まれる。

Q3. ミドルウェアの代表例として、適切なものはどれか。

ア）Windows　イ）Word　ウ）DBMS、Webサーバ、MQTT　エ）Google Chrome

答えを見る ▼

答え: ウ — ミドルウェアの代表例はDBMS、Webサーバ、APIゲートウェイ、MQTT、AIミドルウェアなど。

Q4. DBMS（データベース管理システム）の役割として、適切なものはどれか。

ア）Webページを配信する　イ）データを安全・効率的に保存・検索・更新する　ウ）ロボットを制御する　エ）画像認識を行う

答えを見る ▼

答え: イ — DBMSはデータの永続化、同時アクセス制御、検索高速化、障害復旧を行う。

Q5. Webサーバの代表例として、適切なものはどれか。

ア）MySQL　イ）TensorFlow　ウ）Apache、Nginx　エ）ROS2

答えを見る ▼

答え: ウ — Webサーバの代表例はApache、Nginx、Lighttpd。MySQLはDBMS、TensorFlowはAIミドルウェア。

Q6. IoT向けの軽量メッセージングプロトコルはどれか。

ア）HTTP　イ）FTP　ウ）MQTT　エ）SMTP

答えを見る ▼

答え: ウ — MQTTはIoT向け軽量プロトコル。Pub/Sub方式でデバイス間通信を実現する。

Q7. ロボット制御のためのミドルウェアはどれか。

ア）TensorFlow　イ）ROS2　ウ）Apache　エ）MySQL

答えを見る ▼

答え: イ — ROS2（Robot Operating System 2）はロボット制御のための通信・制御フレームワーク。

Q8. アプリケーションソフトウェアの例として、適切なものはどれか。

ア）Linuxカーネル　イ）FreeRTOS　ウ）Microsoft Office　エ）Nginx

答えを見る ▼

答え: ウ — Microsoft Officeはアプリケーションソフトウェア。他はシステムソフトウェアまたはミドルウェア。

Q9. AIミドルウェアの代表例として、適切なものはどれか。

ア）Apache　イ）TensorFlow、PyTorch　ウ）MQTT　エ）AWS IoT Core

答えを見る ▼

答え: イ — AIミドルウェアの代表例はTensorFlow、PyTorch、ONNX Runtimeなど。

Q10. リアルタイムOS（RTOS）の説明として、適切なものはどれか。

ア）多機能で大規模なアプリケーション向け　イ）タッチ操作に最適化　ウ）リアルタイム性を重視。タスクの優先度が厳密に守られる　エ）オフィス業務向け

答えを見る ▼

答え: ウ — RTOSはリアルタイム性を重視し、タスクの優先度が厳密に守られる。FreeRTOS、VxWorksなど。
