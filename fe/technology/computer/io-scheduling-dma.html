6.5 I/Oスケジューリング

ストレージ（HDD/SSD）などのデバイスは、複数のI/O要求が同時に発生します。しかし、ディスクは一度に1つの要求しか処理できないため、OSは要求の順番を最適化する必要があります。

方式 説明 メリット デメリット
FCFS（先着順） 到着した順に要求を処理する シンプルで実装が簡単。公平 ディスクヘッドの移動が大きくなりやすく、平均待ち時間が長い
SCAN（エレベータ方式） ディスクヘッドを一方向に動かしながら処理し、端まで行ったら反対方向へ戻る ディスクヘッドの移動量が減り、平均待ち時間が短い 端の要求が有利になりやすい

6.6 割り込み（Interrupt）とI/O

割り込み（Interrupt）は、I/Oデバイスが処理完了やイベント発生をCPUに通知する仕組みです。CPUがデバイスを待ち続ける必要がなくなり、効率的なI/Oが可能になります。

項目 内容
割り込みが必要な理由 ボーリング方式（CPUが常に監視）ではCPUが待ち時間で無駄に占有される
割り込みの仕組み デバイス側から「終わったよ」と通知。CPUは他の処理を進めながらI/Oを効率的に扱える
組み込みでの利用例 タイマー割り込み、外部割り込み（ボタン）、通信割り込み（UART/SPI/I2C）、DMA完了割り込み
割り込みの種類 説明 例
ハードウェア割り込み 外部デバイスからの割り込み タイマー割り込み、キーボード入力、ネットワーク受信
ソフトウェア割り込み プログラムが意図的に発生させる割り込み システムコール、例外処理

6.7 DMA（Direct Memory Access）

DMAは、CPUを介さずにデバイスがメモリへ直接データ転送を行う仕組みです。大量データを扱う組み込みシステムでは特に重要なI/O技術です。

項目 内容
DMAを使う理由 通常のI/Oでは「デバイス→CPU→メモリ」の経路でCPUが占有される。DMAではCPUの負荷を大幅に減らせる
DMAの仕組み CPUがDMAコントローラに転送設定 → DMAが転送実行 → 完了時に割り込み → CPUは開始指示と完了通知だけ担当
DMAの利点 CPU負荷軽減、大量データ転送が高速、リアルタイム性の向上
組み込みでのDMA利用例 説明
ADC（アナログ→デジタル変換） 連続サンプリングをDMAでメモリへ自動転送
UART / SPI / I2C通信 受信データをCPU介さずバッファへ格納
カメラモジュールの画像取り込み 大量の画素データを高速転送
PWMキャプチャ 入力パルス幅をDMAで自動記録
オーディオ処理（I2S） 音声データの連続ストリームをDMAで処理

6.8 組み込み・IoT・RTOSにおけるI/O管理

組み込みシステムのI/O管理の特徴

特徴 説明
多様なデバイスを直接制御 センサー、モーター、アクチュエータ、通信モジュールなど、デバイスごとに制御方式が異なる
割り込み駆動が中心 センサー値の変化や通信イベントを即時処理。ボーリングでは遅延が大きくリアルタイム性を満たせない
DMAの積極利用 高速サンプリングや連続データ取得で必須。CPU負荷を軽減
軽量で高速なデバイスドライバ 汎用OSのような複雑な抽象化レイヤは持たない。ハードウェア寄りのシンプルな構造
電源断・省電力を考慮したI/O設計 バッテリ駆動が多く、I/O動作そのものが電力消費に直結。書き込み中の電源断対策が必要

RTOSにおけるI/O管理

特徴 説明
割り込みハンドラが最優先で実行 外部イベントに即応するため
タスクと割り込みの連携 キュー、セマフォ、イベントグループなどを利用。割り込み→タスクへのデータ受け渡しが基本パターン
遅いデバイスはタスク側で処理 通信、ストレージなど時間のかかる処理はタスクで実行。割り込みは「通知だけ」にして遅延を防ぐ

IoTデバイスにおけるI/O特性

特性 説明
省電力のためのI/O制御 デバイスを頻繁にスリープさせる。センサー取得はタイマー駆動や割り込みで必要時のみ実行
無線通信の電力管理 BLE/Wi-Fi/LPWAなどは電力消費が大きい。通信回数・送信タイミングを最適化。バッファリングしてまとめて送信
クラウド連携を前提としたI/O センサー値 → ローカル処理 → 通信 → クラウド。通信I/Oがシステム全体のボトルネックになりやすい

確認問題

Q1. I/Oスケジューリングの目的として、適切なものはどれか。

ア）CPUのクロック周波数を上げる　イ）ディスクヘッドの移動量を減らし平均待ち時間を短くする　ウ）メモリ容量を増やす　エ）ファイルを圧縮する

答えを見る ▼

答え: イ — I/Oスケジューリングの目的は、ディスクヘッドの移動量を減らし、平均待ち時間を短くすること。

Q2. SCAN方式（エレベータ方式）の説明として、適切なものはどれか。

ア）到着した順に処理する方式　イ）ディスクヘッドを一方向に動かしながら処理し、端で折り返す方式　ウ）実行時間が短い要求を優先する方式　エ）ランダムに処理する方式

答えを見る ▼

答え: イ — SCAN方式はディスクヘッドを一方向に動かしながら処理し、端まで行ったら反対方向へ戻るエレベータ方式。

Q3. 割り込みの説明として、最も適切なものはどれか。

ア）CPUが定期的にデバイスの状態を確認すること　イ）デバイスが処理完了をCPUに通知する仕組み　ウ）CPUのクロック周波数を下げる仕組み　エ）メモリを増設する仕組み

答えを見る ▼

答え: イ — 割り込みはデバイスが処理完了やイベント発生をCPUに通知する仕組み。

Q4. DMAの説明として、最も適切なものはどれか。

ア）CPUがデバイスの状態を監視する方式　イ）CPUを介さずにデバイスが直接メモリにアクセスする方式　ウ）データを暗号化して転送する方式　エ）データを圧縮して転送する方式

答えを見る ▼

答え: イ — DMAはCPUを介さずにデバイスが直接メモリにアクセスする方式。大量データ転送に適する。

Q5. DMAの利点として、適切なものはどれか。

ア）CPUの負荷が増える　イ）データ転送が遅くなる　ウ）CPUの負荷が減り、大量データ転送が高速　エ）プログラムが複雑になる

答えを見る ▼

答え: ウ — DMAの利点は「CPU負荷軽減」「大量データ転送が高速」「リアルタイム性の向上」。

Q6. 組み込みシステムのI/O管理の特徴として、本文で挙げられているものはどれか。

ア）仮想記憶を必ず使用する　イ）割り込み駆動が中心、DMAの積極利用、電源断対策　ウ）GUIが必須　エ）大容量メモリが必要

答えを見る ▼

答え: イ — 組み込みI/Oの特徴は「割り込み駆動が中心」「DMAの積極利用」「電源断・省電力を考慮」など。

Q7. FCFS（先着順）の特徴として、適切なものはどれか。

ア）ディスクヘッドの移動量が最も少ない　イ）平均待ち時間が最も短い　ウ）シンプルで実装が簡単だが、移動量が大きくなりやすい　エ）端の要求が有利になりやすい

答えを見る ▼

答え: ウ — FCFSはシンプルで実装が簡単だが、ディスクヘッドの移動が大きくなりやすい。

Q8. RTOSにおける割り込み処理の特徴として、適切なものはどれか。

ア）割り込み処理は最低優先度で実行される　イ）割り込みハンドラは長く複雑に書くのが良い　ウ）割り込みハンドラが最優先で実行され、ISRは短く軽くする　エ）割り込みは使用しない

答え: ウ — RTOSでは割り込みハンドラが最優先で実行され、割り込み遅延を防ぐためISRは短く軽くするのが原則。
