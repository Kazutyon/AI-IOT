主記憶装置・キャッシュメモリ

主記憶装置（RAM/ROM）の種類と役割、キャッシュメモリの階層構造・局所性・書き込み方式を理解する。

用語 主記憶装置 RAMとROM キャッシュメモリ 参照の局所性 問題
ポータル コンピュータ概論 主記憶装置・キャッシュメモリ

このページで覚えること

  • 主記憶装置（メインメモリ）：CPUの作業スペース。プログラムやデータを一時保存
  • RAM（揮発性）：DRAM（PCメインメモリ）とSRAM（キャッシュ用）
  • ROM（不揮発性）：BIOSやファームウェアを格納。マスクROM、PROM、EPROM、EEPROM
  • キャッシュメモリ：CPUと主記憶の速度差を埋める高速メモリ。L1・L2・L3の階層構造
  • 参照の局所性：時間的局所性（繰り返し）と空間的局所性（近いアドレス）
  • キャッシュヒット/ミス：ヒット＝高速、ミス＝低速（主記憶アクセス）
  • 書き込み方式：Write Through（常に同期） vs Write Back（後でまとめて書き込み）

試験に出る用語

用語 意味 試験ポイント
主記憶装置（メインメモリ） コンピュータの作業スペース。CPUが処理するデータやプログラムを一時的に保存 CPUが直接アクセスできる高速な記憶装置。揮発性
RAM Random Access Memory。読み書き可能なメモリ。電源を切ると内容が消える（揮発性） DRAMとSRAMの2種類がある
ROM Read Only Memory。読み出し専用のメモリ。電源を切っても内容を保持（不揮発性） BIOSやファームウェアの格納に使用
DRAM Dynamic RAM。コンデンサに電荷を保持してデータを記憶。リフレッシュが必要 PCやサーバのメインメモリとして一般的。大容量・低コスト
SRAM Static RAM。フリップフロップ回路でデータを保持。リフレッシュ不要で高速 CPUのキャッシュメモリとして利用。高コスト・小容量
キャッシュメモリ CPUと主記憶装置の速度差を埋めるための高速メモリ L1（最速・最小）→ L2 → L3（遅い・大容量）の階層構造
参照の局所性 プログラムが特定のデータや命令に偏ってアクセスする性質 時間的局所性（繰り返し）と空間的局所性（近いアドレス）
キャッシュヒット 必要なデータがキャッシュにあり、高速にアクセスできる状態 主記憶に行かなくてよいので高速
キャッシュミス 必要なデータがキャッシュにない状態 主記憶から読み込むため時間がかかる
Write Through キャッシュに書き込むと同時に主記憶にも書き込む方式 データの整合性が高いが、速度は遅い
Write Back キャッシュ内のデータが追い出されるタイミングでまとめて主記憶に書き込む方式 高速だが、主記憶が最新とは限らない

2.1.2 主記憶装置（メインメモリ）

主記憶装置は、コンピュータの作業スペースにあたる装置です。CPUが処理するデータやプログラムを一時的に保存します。

役割 説明
プログラムの一時保管 実行中のプログラム（命令列）を保持する
データの一時保管 計算や処理に必要なデータを保持する
高速なデータ交換 CPUは主記憶装置から直接データを読み書きすることで処理速度を最大化

RAMとROMの種類と特徴

RAMの細分類

種類 概要 特徴 用途
DRAM コンデンサに電荷を保持してデータを記憶 揮発性、リフレッシュ必要、大容量・低コスト PCメインメモリ、サーバメモリ
SRAM フリップフロップ回路でデータを保持 揮発性、リフレッシュ不要、高速・高コスト CPUキャッシュメモリ、組み込み高速メモリ

ROMの細分類

種類 概要 特徴
マスクROM 製造時に書き込み済みのROM 高速・低コストだが内容を書き換えられない
PROM Programmable ROM。一度だけ書き込み可能なROM 製造後にユーザが書き込み可能。書き込み後は不揮発性
EPROM Erasable PROM。紫外線で消去可能なROM 再書き込み可能。書き換えには専用装置が必要
EEPROM Electrically Erasable PROM。電気的に消去・書き込み可能なROM 再書き込みが容易で、フラッシュメモリとしてPCやスマホに利用

2.1.3 キャッシュメモリ

キャッシュメモリは、CPUが頻繁に利用するデータや命令を一時的に保存する高速メモリです。CPUと主記憶装置の速度差を埋める役割があります。

階層 容量 速度 特徴
L1キャッシュ 数十KB 最速 CPUコア内部に搭載。最も高速だが容量が小さい
L2キャッシュ 数百KB～数MB L1より遅い CPUコア内部またはすぐ近くに配置
L3キャッシュ 数MB～数十MB さらに遅い 複数のコアで共有されることが多い。容量が大きい

参照の局所性とキャッシュの関係

局所性の種類 説明 例
時間的局所性 一度使ったデータは再び使われやすい ループ内で同じ変数を何度も使う、同じ命令を繰り返し実行する
空間的局所性 近くのアドレスのデータが続けて使われやすい 配列を順番に処理する、メモリ上で連続した命令を実行する

キャッシュの書き込み方式

方式 動作 メリット デメリット 用途
Write Through キャッシュに書き込むと同時に主記憶にも書き込む データの整合性が高い 速度が遅い 金融システム、組み込み機器、マルチコアCPUでの共有メモリ
Write Back キャッシュ内のデータが追い出されるタイミングでまとめて主記憶に書き込む 高速 主記憶が最新状態とは限らない PC・サーバのCPU、ゲーム機、高速処理が必要なアプリケーション

確認問題

Q1. DRAMとSRAMの違いに関する説明として、適切なものはどれか。

ア）DRAMは不揮発性、SRAMは揮発性　イ）DRAMはリフレッシュが必要、SRAMは不要　ウ）DRAMは高速・高コスト、SRAMは低速・低コスト　エ）DRAMはキャッシュ用、SRAMは主記憶用

答えを見る ▼

答え: イ — DRAMはコンデンサに電荷を保持するためリフレッシュが必要。SRAMはフリップフロップ回路で保持するため不要。

Q2. 主記憶装置の役割として、適切でないものはどれか。

ア）実行中のプログラムを保持する　イ）計算に必要なデータを保持する　ウ）データを永続的に保存する　エ）CPUと高速にデータ交換する

答えを見る ▼

答え: ウ — 主記憶装置は揮発性メモリであり、電源を切ると内容が消える。永続的な保存は外部記憶装置（HDD/SSD）の役割。

Q3. BIOSやファームウェアを格納するのに適したメモリはどれか。

ア）DRAM　イ）SRAM　ウ）ROM　エ）キャッシュメモリ

答えを見る ▼

答え: ウ — ROMは不揮発性で電源を切っても内容を保持するため、BIOSやファームウェアの格納に適している。

Q4. キャッシュメモリの階層構造で、最も高速なものはどれか。

ア）L1キャッシュ　イ）L2キャッシュ　ウ）L3キャッシュ　エ）主記憶装置

答えを見る ▼

答え: ア — L1キャッシュはCPUコア内部に搭載され、最も高速だが容量が小さい。

Q5. 参照の局所性のうち「時間的局所性」の説明として、適切なものはどれか。

ア）近くのアドレスのデータが続けて使われやすい　イ）一度使ったデータは再び使われやすい　ウ）データがランダムにアクセスされる　エ）すべてのデータが均等に使われる

答えを見る ▼

答え: イ — 時間的局所性は「一度使ったデータは再び使われやすい」性質。空間的局所性は「近くのアドレスのデータが続けて使われやすい」。

Q6. キャッシュミスが発生する原因として、本文で挙げられているものはどれか。

ア）CPUのクロック周波数が低い　イ）初回アクセス、容量不足、データの置き換え　ウ）メモリ容量が大きすぎる　エ）OSのバージョンが古い

答えを見る ▼

答え: イ — キャッシュミスの原因は「初回アクセス」「容量不足」「データの置き換え」。

Q7. Write Back方式の特徴として、適切なものはどれか。

ア）キャッシュに書き込むと同時に主記憶にも書き込む　イ）主記憶への書き込みが減り高速だが、主記憶が最新とは限らない　ウ）データの整合性が最も高い　エ）金融システムでよく使われる

答えを見る ▼

答え: イ — Write Backはキャッシュ内のデータが追い出されるタイミングでまとめて書き込むため高速だが、主記憶が最新とは限らない。

Q8. 空間的局所性の例として、適切なものはどれか。

ア）ループ内で同じ変数を何度も使う　イ）配列を順番に処理する　ウ）関数を呼び出す　エ）エラー処理を行う

答えを見る ▼

答え: イ — 空間的局所性は「近くのアドレスのデータが続けて使われやすい」。配列の順次処理はその典型例。

Q9. EEPROMの特徴として、適切なものはどれか。

ア）紫外線で消去する　イ）一度だけ書き込める　ウ）電気的に消去・書き込みが可能　エ）製造時に書き込み済み

答えを見る ▼

答え: ウ — EEPROMは電気的に消去・書き込みが可能。フラッシュメモリとしてPCやスマホに利用される。

Q10. PCのメインメモリとして一般的に使用されるメモリはどれか。

ア）SRAM　イ）DRAM　ウ）マスクROM　エ）EPROM

答えを見る ▼

答え: イ — PCのメインメモリにはDRAM（特にDDR4/DDR5）が使用される。SRAMはキャッシュメモリに使用される。
