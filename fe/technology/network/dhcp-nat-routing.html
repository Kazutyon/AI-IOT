DHCP（Dynamic Host Configuration Protocol）

ネットワークに機器を接続した瞬間、IPアドレスはどのように決まるのか。DHCPは、LAN内の機器にIPアドレス・サブネットマスク・デフォルトゲートウェイ・DNSサーバーなどの情報を自動的に割り当てるプロトコルである。

手動設定では設定ミスや重複が起きやすく、大規模ネットワークでは管理が破綻する。DHCPサーバー（通常はルーターや高機能スイッチが兼任）がアドレスプールを一元管理することで、接続するだけで通信可能になる。

DHCPのDORAシーケンス

DHCPクライアントとサーバー間の4ステップのやり取りをDORAと呼ぶ。午前試験では「正しい順序はどれか」という形式で頻出。

  1. DHCP Discover（クライアント→ブロードキャスト）: DHCPサーバーを探索。宛先は255.255.255.255
  2. DHCP Offer（サーバー→クライアント）: 利用可能なIPアドレスを提案
  3. DHCP Request（クライアント→サーバー）: 特定のサーバーに正式に割り当てを要求（複数Offerが来た場合も1つだけ選択）
  4. DHCP Ack（サーバー→クライアント）: IPアドレス確定。サブネットマスク・デフォルトゲートウェイ・DNSもここで確定

頻出：「DORAの正しい順序は？」→ Discover→Offer→Request→Ack。「DHCPクライアントが最初に送るメッセージは？」→ DHCP Discover（ブロードキャスト）

リース期間

DHCPで割り当てられるIPアドレスには有効期限（リース期間）がある。期限が切れると再割り当てが発生するため、IPアドレスを効率的に再利用できる。大規模ネットワークや一時接続が多い環境でDHCPが有効な理由のひとつ。

NAT / NAPT

LAN内の機器にはプライベートIPアドレス（例: 192.168.0.x）が割り当てられる。しかしプライベートIPはインターネット上では使えない。インターネットに出るには、グローバルIPアドレスに変換する仕組みが必要。これがNAT/NAPTである。

NAT（Network Address Translation）

  • プライベートIPアドレスとグローバルIPアドレスを1対1で変換
  • 変換表（プライベートIP⇔グローバルIPの対応表）で管理
  • 内部機器ごとに異なるグローバルIPが必要→企業ネットワーク向け

NAPT（Network Address Port Translation）

  • NATに加えてTCP/UDPの送信元ポート番号も変換に使用
  • 1つのグローバルIPで複数の内部機器を同時接続可能
  • ポート番号で「どの機器からの通信か」を識別
  • IPマスカレードとも呼ばれ、家庭用ルーターの標準機能
  • 返信パケットもポート番号を見て適切な内部機器に届ける

頻出：「NATとNAPTの違いは？」→ NATはIPのみ1対1変換、NAPTはIPとポート番号を使い多対1変換。「家庭用ルーターで使われる技術は？」→ NAPT（IPマスカレード）

ルーティング

インターネットは世界中のLANを相互接続した巨大なネットワーク。宛先まで適切な経路を選ぶ技術がルーティングである。ルーターは受信パケットを調べ、自分が管理するLAN宛てなら直接届け、それ以外は別のルーターに転送する。

経路選択の指標をメトリック（ホップ数、回線速度、料金など）と呼び、経路情報はルーティングテーブルに管理される。

ルーティングプロトコルの分類（IGP / EGP）

ルーター同士が経路情報を交換する通信規則をルーティングプロトコルといい、適用範囲によってIGPとEGPに大別される。

IGP（Interior Gateway Protocol）：同一AS（自律システム）内で使う

プロトコル分類メトリック特徴
RIPディスタンスベクタ型ホップ数のみ最大15ホップ。小規模向け。シンプルだが大規模では非効率
OSPFリンクステート型回線速度（コスト）ネットワーク全体の状態を把握。大規模向け。信頼性高い
EIGRPハイブリッド型複合Cisco独自。RIPとOSPFの利点を組み合わせ

EGP（Exterior Gateway Protocol）：異なるAS間で使う

プロトコル分類特徴
BGPパスベクタ型AS番号のリストで経路決定。インターネット全体の標準。経済性・ポリシーを考慮
旧EGPディスタンスベクタ型現在はほぼ使われていない。BGPに置き換え済み

頻出：「RIPの分類は？」→ ディスタンスベクタ型（ホップ数）。「OSPFの分類は？」→ リンクステート型（大規模向け）。「インターネット全体の標準プロトコルは？」→ BGP（EGP・パスベクタ型）

デフォルトゲートウェイ

ルーターはルーティングテーブルに経路情報があるパケットは転送できるが、経路情報のないパケット（未知のパケット）は通常破棄してしまう。

デフォルトゲートウェイは、こうした「どの経路を通れば届くか分からないパケット」のデフォルトの転送先となるルーターである。LAN内の各機器に、そのLANを管理するルーターのIPアドレスをデフォルトゲートウェイとして設定することで、インターネット上のWebサイトへのアクセスなど、LAN外への通信が可能になる。

  • デフォルトゲートウェイ設定あり → 未知のパケットはデフォルトゲートウェイへ転送 → 上位ネットワークへ
  • デフォルトゲートウェイ設定なし → 未知のパケットは破棄 → 通信不能

頻出：「デフォルトゲートウェイが未設定の場合、未知パケットは？」→ 破棄される。「デフォルトゲートウェイに指定するIPアドレスは？」→ そのLANを管理するルーターのIPアドレス

間違えやすいポイント

  • DORAの順序：「Discover→Offer→Request→Ack」が正しい。「Requestが先」や「OfferとRequestが逆」という選択肢に注意。DORA（ドラ）と覚える。
  • NATとNAPTの混同：NATは「1対1」、NAPTは「多対1でポート番号を使う」。NAPT＝IPマスカレード、家庭用ルーター＝NAPTと紐付ける。
  • RIPとOSPFの取り違え：「ディスタンスベクタ＝RIP」「リンクステート＝OSPF」の組み合わせを暗記。RIPはホップ数のみ（最大15）、OSPFは回線速度も考慮。
  • IGPとEGPの境界：IGP＝AS内、EGP＝AS間。BGPはEGPなので「AS間」。インターネット全体＝BGP。
  • デフォルトゲートウェイ未設定の結果：「パケットは破棄される」が正解。「自動で別ルーターに転送」や「エラーが返る」ではない。
  • DHCP Discoverの宛先：宛先IPアドレスは「255.255.255.255」（ブロードキャスト）。FF:FF:FF:FF:FF:FFはMACアドレスのブロードキャストで混同しやすい。

確認問題

Q1. DHCPの説明として適切なものはどれか。
A) ドメイン名をIPアドレスに変換するプロトコルである　B) IPアドレスを自動的に割り当てるプロトコルである　C) IPアドレスからMACアドレスを求めるプロトコルである　D) プライベートIPアドレスをグローバルIPアドレスに変換するプロトコルである
答えを見る ▼

答え: B。DHCP（Dynamic Host Configuration Protocol）はIPアドレスを自動割り当てする。AはDNS、CはARP、DはNAT/NAPT。

Q2. NAT（Network Address Translation）の説明として適切なものはどれか。
A) MACアドレスをIPアドレスに変換する　B) プライベートIPアドレスとグローバルIPアドレスを1対1で変換する　C) 1つのグローバルIPアドレスで複数の内部機器をポート番号で区別する　D) 経路情報を管理するルーティングテーブルを作成する
答えを見る ▼

答え: B。NATはプライベートIPアドレスをグローバルIPアドレスに1対1で変換する。CはNAPT（IPマスカレード）の説明。

Q3. ルーティングテーブルに経路が存在しないパケットの転送先として指定されるものはどれか。
A) DNSサーバー　B) DHCPサーバー　C) デフォルトゲートウェイ　D) プロキシサーバー
答えを見る ▼

答え: C。デフォルトゲートウェイは、未知の宛先パケットの転送先となるルーター。設定されていない場合、パケットは破棄される。

Q4. DHCPにおけるメッセージのやり取りの正しい順序はどれか。
A) Request → Discover → Offer → Ack　B) Discover → Offer → Request → Ack　C) Offer → Discover → Ack → Request　D) Discover → Request → Offer → Ack
答えを見る ▼

答え: B。DORA（Discover→Offer→Request→Ack）の順。まずクライアントがサーバーを探索（Discover）、サーバーが提案（Offer）、クライアントが要求（Request）、サーバーが確定（Ack）。

Q5. NAPT（IPマスカレード）がNATと異なる点として正しいものはどれか。
A) プライベートIPアドレスを使わない　B) 1つのグローバルIPアドレスで複数の内部機器を同時にインターネット接続できる　C) MACアドレスも変換に使用する　D) ルーティングプロトコルを自動選択する
答えを見る ▼

答え: B。NAPTはTCP/UDPの送信元ポート番号で内部機器を区別するため、1つのグローバルIPアドレスで複数機器の同時接続が可能。NATは1対1変換のためグローバルIPが複数必要。

Q6. ディスタンスベクタ型のルーティングプロトコルはどれか。
A) OSPF　B) BGP　C) RIP　D) EIGRP
答えを見る ▼

答え: C。RIP（Routing Information Protocol）はホップ数（経由ルーター数）で最短経路を決めるディスタンスベクタ型。OSPFはリンクステート型、BGPはパスベクタ型（EGP）、EIGRPはハイブリッド型。

Q7. インターネット全体で標準的に使われているルーティングプロトコルはどれか。
A) RIP　B) OSPF　C) EIGRP　D) BGP
答えを見る ▼

答え: D。BGP（Border Gateway Protocol）はAS間のルーティングに使われるEGPで、インターネット全体の標準プロトコル。AS番号のリストで経路を決定し、経済性やポリシーも考慮可能。

Q8. DHCP Discoverメッセージの宛先として使われるアドレスはどれか。
A) 192.168.0.1　B) FF:FF:FF:FF:FF:FF　C) 255.255.255.255　D) 127.0.0.1
答えを見る ▼

答え: C。DHCP DiscoverはブロードキャストでDHCPサーバーを探索するため、宛先IPアドレスは255.255.255.255（ブロードキャストアドレス）。BはMACアドレスのブロードキャスト、Dはループバックアドレス。

Q9. IGP（Interior Gateway Protocol）の説明として正しいものはどれか。
A) 異なるAS間で経路情報を交換するプロトコルである　B) インターネット全体の経路制御に使われる標準プロトコルである　C) 同一AS内でルーター同士が経路情報を交換するプロトコルである　D) プライベートIPアドレスとグローバルIPアドレスを変換するプロトコルである
答えを見る ▼

答え: C。IGPは同一AS（自律システム）内のルーティングに使う。AとBはEGP（BGP）の説明。DはNAT/NAPT。

Q10. デフォルトゲートウェイが設定されていない場合、ルーティングテーブルに存在しない宛先のパケットはどうなるか。
A) 自動的に最も近いルーターに転送される　B) 送信元にエラーメッセージが返される　C) パケットは破棄される　D) ブロードキャストで全ルーターに送信される
答えを見る ▼

答え: C。デフォルトゲートウェイが未設定の場合、ルーターは未知の宛先パケットを破棄する。デフォルトゲートウェイを設定することで、こうしたパケットも適切に転送できるようになる。
