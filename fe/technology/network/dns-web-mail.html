インターネットの基本

インターネットは、企業LANやISP、モバイルネットワークなど様々なネットワークを相互接続する広域ネットワークです。起源は1967年のARPANET計画で、1969年にはアメリカの4大学・研究機関をつなぐネットワークが稼働しました。ARPANETの研究成果としてTCP/IPプロトコルも開発されました。

ルーターの役割（再掲）

  • LAN内の通信は外部に出さない
  • 自分のLAN宛でない通信は内部に入れない
  • 自LAN宛でない通信はルーティングテーブルに従って適切に転送

ISP（インターネットサービスプロバイダ）：個人や企業にインターネット接続回線を提供する業者。代表例：NTT、KDDI、ソフトバンク。エッジルーター・ファイアウォール・ISPの3つが役割を分担して安全な通信を実現します。

メールのしくみ

プロトコル用途ポート特徴
SMTPメール送信25 / 587送信専用。サーバー間の転送にも使われる
POP3メール受信110サーバーから端末にダウンロード。ダウンロード後はサーバーから削除されるのが基本
IMAPメール受信143サーバー上でメールを管理。複数端末から同じメールボックスを参照可能

SMTPで送信、POP3またはIMAPで受信という役割分担が基本です。IMAPはサーバーにメールを残すため、スマホとPCで同じメールを読む場合に適しています。

Webのしくみ

HTTP（Hypertext Transfer Protocol）

Webページを転送するプロトコル。ステートレス（状態を持たない）が特徴で、各リクエストは独立して処理されます。ポート番号は80。

HTTPS（HTTP Secure）

HTTPをSSL/TLSで暗号化したもの。ポート番号は443。通信の盗聴・改ざん・なりすましを防止します。

URLとURI

URL（Uniform Resource Locator）は「スキーム://ホスト/パス」の形式でリソースの場所を示します。例：https://www.example.com/index.html

  • スキーム：http / https / ftp など
  • ホスト：ドメイン名またはIPアドレス
  • パス：サーバー上のファイル位置

DNS（Domain Name System）

DNSはドメイン名をIPアドレスに変換するシステムです。人間が覚えやすい名前（例：www.example.com）をコンピューターが使うIPアドレスに変換します。

名前解決の流れ

  1. クライアントがDNSキャッシュサーバーに問合せ
  2. キャッシュがあれば即応答、なければルートDNSサーバーへ
  3. ルート→TLD（.comなど）→権威DNSサーバーと順に問合せ
  4. 解決したIPアドレスをクライアントに返す

再帰問合せ：DNSサーバーが代理で全部調べて結果だけ返す。反復問合せ：DNSサーバーが「次はここに聞け」と順に案内する。

AjaxとREST API

Ajax（Asynchronous JavaScript + XML）

ページ全体を再読み込みせず、非同期通信で必要なデータだけを取得して画面を部分的に更新する技術。Google Mapsのスクロールで新しい地図が読み込まれるのが代表例。

REST API

HTTPメソッド（GET/POST/PUT/DELETE）でリソースを操作するWeb APIの設計スタイル。URLでリソースを指定し、JSON形式でデータをやり取りするのが一般的です。

間違えやすいポイント

  • SMTPは送信専用、POP3/IMAPは受信専用：送信と受信でプロトコルが分かれている。
  • POP3とIMAPの違い：POP3=ダウンロード型（サーバーから消える）、IMAP=サーバー管理型（複数端末で同期）。
  • HTTPは80番、HTTPSは443番：ポート番号の暗記必須。
  • HTTPはステートレス：状態を保持しない。セッション管理はCookieなどで実現。
  • DNSの再帰問合せと反復問合せ：再帰=全部任せる、反復=自分で順に聞く。

確認問題

Q1. メール送信に使われるプロトコルはどれか。

ア）POP3
イ）IMAP
ウ）SMTP
エ）HTTP

答えを見る ▼

答え: ウ — SMTPはメール送信用。POP3とIMAPは受信用。

Q2. HTTPS通信で使用されるポート番号はどれか。

ア）25
イ）80
ウ）110
エ）443

答えを見る ▼

答え: エ — HTTP=80、HTTPS=443。SMTP=25、POP3=110。

Q3. DNSの役割として正しいものはどれか。

ア）メールを暗号化して送信する
イ）ドメイン名をIPアドレスに変換する
ウ）Webページの表示速度を向上させる
エ）ファイルを圧縮して転送する

答えを見る ▼

答え: イ — DNSはDomain Name System。ドメイン名→IPアドレスの名前解決を行う。

Q4. IMAPとPOP3の違いとして正しいものはどれか。

ア）IMAPはメール送信に使われる
イ）POP3はサーバー上にメールを残す
ウ）IMAPはサーバー上でメールを管理し、複数端末から参照できる
エ）POP3とIMAPに機能差はない

答えを見る ▼

答え: ウ — IMAPはサーバー管理型で複数端末同期可能。POP3は基本的にダウンロード後削除。

Q5. HTTPの特徴として正しいものはどれか。

ア）ステートフル（状態を保持する）
イ）ステートレス（状態を保持しない）
ウ）メール送信に使われる
エ）通信が常に暗号化される

答えを見る ▼

答え: イ — HTTPはステートレス。各リクエストは独立。セッション管理はCookie等で実現。

Q6. Ajaxの説明として正しいものはどれか。

ア）ページ全体を再読み込みして画面を更新する
イ）非同期通信で必要なデータだけを取得し、画面を部分的に更新する
ウ）サーバー側でHTMLを生成して返す
エ）メールを暗号化する技術

答えを見る ▼

答え: イ — Ajaxは非同期通信で部分更新。Google Mapsなどが代表例。

Q7. URLの「スキーム」部分の例として正しいものはどれか。

ア）www.example.com
イ）/index.html
ウ）https
エ）#section1

答えを見る ▼

答え: ウ — httpsがスキーム。アはホスト、イはパス、エはフラグメント。

Q8. インターネットの起源となったプロジェクトはどれか。

ア）ARPANET
イ）イーサネット
ウ）ワールドワイドウェブ
エ）TCP/IP

答えを見る ▼

答え: ア — ARPANET（1969年稼働）がインターネットの起源。TCP/IPもARPANETで開発された。
