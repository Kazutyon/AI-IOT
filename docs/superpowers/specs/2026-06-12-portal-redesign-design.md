# FE試験対策サイト ポータルリデザイン 設計書

作成日: 2026-06-12  
対象リポジトリ: AI-IOT  
開発ブランチ: `redesign`

---

## 背景・目的

- 元々は自習用だったが、クラスメート（AI・IoT DXエンジニア養成科）にも公開することになった
- スマートフォンでの復習・確認用途がメイン
- 静的サイト（GitHub Pages）のまま機能を追加し、使いやすさと見栄えを向上させる

---

## アーキテクチャ

静的サイト（GitHub Pages）のまま構築。バックエンド・サーバー不要。

```
assets/
  style.css          ← 既存（拡張）
  app.js             ← 新規：全JS機能（進捗・検索・今日の一問）
  search-index.json  ← 新規：全ページ用語データ（手動 or スクリプト生成）
index.html           ← ポータル全面改修
search.html          ← 新規：用語検索ページ
progress.html        ← 新規：学習進捗ページ
quiz-data.json       ← 新規：全クイズ問題データ
```

ユーザーデータはすべて `localStorage` に保存。別デバイスへの引き継ぎは対象外（クラスメート用途では許容）。

---

## フェーズ計画

| Phase | 内容 | 規模 |
|---|---|---|
| 1 | ポータル（index.html）全面改修 + ボトムナビ | 中 |
| 2 | 今日の一問 + 頻出度バッジ | 小 |
| 3 | 用語検索（search.html） | 中 |
| 4 | 学習進捗保存 + 最近の履歴（progress.html） | 中 |
| 5 | よくある質問 + おすすめ学習コース | 小 |

AI先生は静的サイトの範囲外のため対象外。

## Phase 1：ポータルレイアウト

### index.html 構成

```
┌─────────────────────────────┐
│ ヘッダー（現行の濃紺、維持）    │
└─────────────────────────────┘
│ このサイトについて（1〜2行）    │
│ アイコン4つ（特徴紹介）        │
├─────────────────────────────┤
│ 学習コンテンツ（カードグリッド）  │
│ ★頻出度バッジ付き             │
├─────────────────────────────┤
│ おすすめ学習コース             │
│ 1→2→3→4→5 ステップ表示      │
├─────────────────────────────┤
│ 今日の一問                   │
│ ヒント表示ボタン付き           │
├─────────────────────────────┤
│ 最近見たページ（localStorage） │
└─────────────────────────────┘
│ ボトムナビ（スマホのみ）        │
│ 🏠ホーム / 🔍検索 / 📊進捗 / 📝メモ │
└─────────────────────────────┘
```

### ボトムナビ
- CSS `@media (max-width: 768px)` でスマホのみ表示
- fixed bottom、高さ56px
- タブ：ホーム / 検索 / 進捗 / メモ

---

## Phase 2：今日の一問 + 頻出度バッジ

### 今日の一問
- 全ページのクイズ問題を `quiz-data.json` にまとめる
- `new Date()` の年月日をシードに問題をランダム選択（毎日自動で変わる）
- 「答えを見る」ボタンで解答表示
- app.js に実装

### 頻出度バッジ
- ポータルカードとフォルダindex.htmlに ★★★ / ★★ / ★ を静的追加
- FE午前の出題頻度を基準に手動設定
- CSSクラス `.badge--high` / `.badge--mid` / `.badge--low` で色分け

---

## Phase 3：用語検索（search.html）

- 全ページの用語・意味・ページURLを `search-index.json` に格納
- 入力のたびにJSでインクリメンタル検索（サーバー不要）
- 結果クリックで該当ページに遷移
- 人気ワードは静的リストとして手動管理

---

## Phase 4：学習進捗（progress.html）

- 各ページ閲覧時に `localStorage` にページIDを記録（app.js）
- progress.htmlでセクション別既読率を表示（棒グラフ）
- 「リセット」ボタンあり
- 最近見たページ5件もlocalStorageから取得してポータルに表示

---

## Phase 5：よくある質問 + おすすめコース

- よくある質問：静的HTML（`<details>/<summary>` で開閉）
- おすすめ学習コース：静的HTML、推奨順に番号付きステップ表示

---

## 開発手順

1. `redesign` ブランチを切る
2. Phase 1〜5 を順番に実装・ローカル確認
3. 問題なければ main にマージ → GitHub Pages に反映

---

## 対象外

- AI先生（バックエンドAPI必要）
- デバイス間の進捗同期
- ユーザー認証・ログイン
