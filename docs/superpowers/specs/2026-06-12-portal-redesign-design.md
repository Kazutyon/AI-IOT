# FE試験対策サイト ポータルリデザイン 設計書

作成日: 2026-06-12  
最終更新: 2026-06-12（かずさん修正反映）  
対象リポジトリ: AI-IOT  
開発ブランチ: `redesign`

---

## 背景・目的

- 元々は自習用だったが、クラスメート（AI・IoT DXエンジニア養成科）にも公開することになった
- スマートフォンでの復習・確認用途がメイン
- 静的サイト（GitHub Pages）のまま機能を追加し、使いやすさと見栄えを向上させる

## 開発方針（最優先）

> **スマホ表示を最優先。PC表示はスマホレイアウトを広げる程度でよい。**  
> **新機能より、読みやすさ・迷わなさ・軽さを優先すること。**

---

## アーキテクチャ

静的サイト（GitHub Pages）のまま構築。バックエンド・サーバー不要。

```
assets/
  style.css          ← 既存（拡張）
  app.js             ← 新規：全JS機能（進捗・検索・今日の一問）
  search-index.json  ← 新規：手動作成の用語データ（20〜50語からスタート）
index.html           ← ポータル全面改修
search.html          ← 新規：用語検索ページ
progress.html        ← 新規：学習進捗ページ
quiz-data.json       ← 新規：全クイズ問題データ
```

ユーザーデータはすべて `localStorage` に保存。別デバイスへの引き継ぎは対象外（クラスメート用途では許容）。

---

## フェーズ計画

| Phase | 内容 | 規模 |
|---|---|---|
| 1 | ポータル改修（対象者・免責・カード・ボトムナビ）| 小〜中 |
| 2 | 今日の一問 + 頻出度バッジ | 小 |
| 3 | 用語検索（search.html）手動20〜50語 | 小〜中 |
| 4 | 学習進捗保存 + 更新履歴（progress.html） | 中 |
| 5 | よくある質問 + おすすめ学習コース | 小 |

AI先生・メモ機能は対象外（静的サイトの範囲外 or 後回し）。

```

│ 復習ノートです。」             │
├─────────────────────────────┤
│ 学習コンテンツ（カードグリッド）  │
│ ★頻出度バッジ付き             │
├─────────────────────────────┤
│ おすすめ学習コース             │
│ 1→2→3→4→5 ステップ表示      │
├─────────────────────────────┤
│ 今日の一問                   │
```

### ボトムナビ
- CSS `@media (max-width: 768px)` でスマホのみ表示
- fixed bottom、高さ56px
- タブ：ホーム / 検索 / 進捗 / 更新履歴（メモは後回し）

---

## Phase 2：今日の一問 + 頻出度バッジ

### 今日の一問
- 全ページのクイズ問題を `quiz-data.json` にまとめる
- `new Date()` の年月日をシードに問題をランダム選択（毎日自動で変わる）
- 「答えを見る」ボタンで解答表示
- app.js に実装

### 頻出度バッジ
- ポータルカードとフォルダ index.html に ★★★ / ★★ / ★ を静的追加
- FE午前の出題頻度を基準に手動設定
- CSSクラス `.badge--high` / `.badge--mid` / `.badge--low` で色分け

---

## Phase 3：用語検索（search.html）

- 最初は手動で20〜50語だけ `search-index.json` に登録（全自動生成は後回し）
- フォーマット：
```json
[
  {
    "term": "DFS",
    "summary": "深さ優先探索。行けるところまで深く探索する方法。",
    "url": "fe/technology/algorithm/algo-tree-search.html",
    "tags": ["アルゴリズム", "探索"]
  }
]
```
- 入力のたびにJSでインクリメンタル検索（サーバー不要）
- 結果クリックで該当ページに遷移

---

## Phase 4：学習進捗（progress.html）

### 進捗の記録方法
- **「このページを学習済みにする」ボタン方式**（ページ閲覧だけで完了にしない）
- 各ページ下部に配置：
```html
<button data-complete="algorithm-dfs">このページを学習済みにする</button>
```
- クリックで `localStorage` に記録、ボタンが「✅ 学習済み」に変わる

### progress.html の表示
- セクション別既読率を棒グラフで表示
- 「リセット」ボタンあり

### 更新履歴（ボトムナビ4番目）
- 最近「学習済み」にしたページ5件を時系列表示
- localStorage から取得

---

## Phase 5：よくある質問 + おすすめコース

- よくある質問：静的HTML（`<details>/<summary>` で開閉）
- おすすめ学習コース：静的HTML、推奨順に番号付きステップ表示

---

## 開発手順

1. `redesign` ブランチを切る
2. Phase 1〜5 を順番に実装・ローカル確認
3. 問題なければ main にマージ → GitHub Pages に反映
